Esta guia cubre instrucciones detalladas de configuracion para cada proveedor soportado por Claudex, incluyendo proveedores de clave API, suscripciones OAuth, plataformas en la nube y modelos locales.

## Proveedores con clave API

### Anthropic

1. Registrate en [console.anthropic.com](https://console.anthropic.com/)
2. Navega a **API Keys** y crea una nueva clave (formato: `sk-ant-*`)

```toml
[[profiles]]
name = "anthropic"
provider_type = "DirectAnthropic"
base_url = "https://api.anthropic.com"
api_key = "sk-ant-..."
default_model = "claude-sonnet-4-20250514"
```

```bash
claudex profile test anthropic
```

## Plataformas en la nube

### Azure OpenAI

1. Crea un recurso Azure OpenAI en el [Azure Portal](https://portal.azure.com/)
2. Despliega un modelo y anota el nombre del recurso y del despliegue
3. Obtener tu clave API en **Keys and Endpoint**

```toml
[[profiles]]
name = "azure-openai"
provider_type = "OpenAICompatible"
base_url = "https://YOUR_RESOURCE.openai.azure.com/openai/deployments/YOUR_DEPLOYMENT"
api_key = "YOUR_AZURE_KEY"
default_model = "gpt-4o"

[profiles.query_params]
api-version = "2024-12-01-preview"
```

```bash
claudex profile test azure-openai
```

<Aside type="note">
  Azure OpenAI se detecta automaticamente por `base_url` que contenga `openai.azure.com`. La autenticacion usa la cabecera `api-key` en lugar de `Authorization: Bearer`. El parametro de consulta `api-version` es obligatorio.
</Aside>

### Google Vertex AI

1. Habilita la API de Vertex AI en tu proyecto GCP
2. Genera un token de acceso con `gcloud auth print-access-token`

```toml
[[profiles]]
name = "vertex-ai"
provider_type = "DirectAnthropic"
base_url = "https://us-east5-aiplatform.googleapis.com/v1/projects/YOUR_PROJECT/locations/us-east5/publishers/anthropic/models"
api_key = "YOUR_GCLOUD_TOKEN"
default_model = "claude-sonnet-4@20250514"
```

```bash
claudex profile test vertex-ai
```

<Aside type="caution">
  La salida de `gcloud auth print-access-token` expira despues de ~1 hora. Para uso en produccion, considera una cuenta de servicio con automatizacion de refresco de tokens.
</Aside>

### AWS Bedrock (via LiteLLM)

AWS Bedrock se soporta a traves de un proxy LiteLLM:

1. Instala LiteLLM: `pip install litellm`
2. Inicia el proxy:

```bash
litellm --model bedrock/anthropic.claude-sonnet-4-20250514-v2:0
```

3. Configura Claudex:

```toml
[[profiles]]
name = "bedrock"
provider_type = "OpenAICompatible"
base_url = "http://localhost:4000/v1"
api_key = "sk-litellm"
default_model = "bedrock/anthropic.claude-sonnet-4-20250514-v2:0"
```

## Suscripciones OAuth

La autenticacion OAuth te permite usar suscripciones existentes de proveedores (ChatGPT Plus, Claude Max, etc.) en lugar de claves API separadas.

### Claude Max / Pro

Usa tu suscripcion existente de Claude a traves de la sesion OAuth nativa de Claude Code.

**Requisitos previos**: Claude Code instalado e iniciado sesion (el comando `claude` funciona normalmente)

```toml
[[profiles]]
name = "claude-max"
provider_type = "DirectAnthropic"
base_url = "https://api.claude.ai"
default_model = "claude-sonnet-4-20250514"
auth_type = "oauth"
oauth_provider = "claude"

[profiles.models]
haiku = "claude-haiku-4-20250514"
sonnet = "claude-sonnet-4-20250514"
opus = "claude-opus-4-20250514"
```

<Aside type="note">
  Los perfiles OAuth de Claude usan un modo especial de gateway auth. Claudex omite el proxy y deja que Claude Code use su propio OAuth directamente. No se necesita un paso separado de `claudex auth login`.
</Aside>

### ChatGPT Plus / Codex CLI

Usa tu suscripcion de ChatGPT Plus o Pro mediante el token del Codex CLI.

**Requisitos previos**: Instala [Codex CLI](https://github.com/openai/codex) y autenticate:

```bash
# Instalar Codex CLI
npm install -g @openai/codex

# Autenticar (abre el navegador)
codex auth
```

Luego configura Claudex:

```toml
[[profiles]]
name = "codex-sub"
provider_type = "OpenAIResponses"
base_url = "https://chatgpt.com/backend-api/codex"
default_model = "gpt-5.3-codex"
auth_type = "oauth"
oauth_provider = "openai"

[profiles.models]
haiku = "codex-mini-latest"
sonnet = "gpt-5.3-codex"
opus = "gpt-5.3-codex"
```

```bash
# Leer token desde Codex CLI
claudex auth login openai --profile codex-sub

# Verificar
claudex auth status

# Ejecutar
claudex run codex-sub
```

<Aside type="tip">
  Modelos soportados en el endpoint Codex: `gpt-5.3-codex`, `gpt-5.1-codex`, `codex-mini-latest`. Ten en cuenta que `gpt-4o` NO esta soportado en el endpoint Codex de `chatgpt.com`. `strip_params = "auto"` detectara automaticamente este endpoint y eliminara `temperature`/`top_p`/`top_k`.
</Aside>

### GitLab Duo

Usa tu suscripcion de GitLab Duo mediante un Personal Access Token.

```toml
[[profiles]]
name = "gitlab-duo"
provider_type = "OpenAICompatible"
base_url = "https://gitlab.com/api/v4/ai/llm/proxy"
default_model = "claude-sonnet-4-20250514"
auth_type = "oauth"
oauth_provider = "gitlab"
```

```bash
# Establece tu token de GitLab como variable de entorno
export GITLAB_TOKEN=glpat-...

# Login
claudex auth login gitlab --profile gitlab-duo

# Verificar
claudex auth status
```

<Aside type="note">
  GitLab Duo requiere una variable de entorno `GITLAB_TOKEN` con un Personal Access Token que tenga el scope `ai_features`. Para instancias GitLab autogestionadas, actualiza `base_url` correspondentemente.
</Aside>

## Modelos locales

### Ollama

1. Instala Ollama: [ollama.com](https://ollama.com/)
2. Descarga un modelo:

```bash
ollama pull qwen2.5:72b
```

3. Configura Claudex:

```toml
[[profiles]]
name = "local-qwen"
provider_type = "OpenAICompatible"
base_url = "http://localhost:11434/v1"
api_key = ""
default_model = "qwen2.5:72b"
```

```bash
claudex profile test local-qwen
```

<Aside type="note">
  Ollama no requiere clave API. Deja `api_key` vacio u omitelo.
</Aside>

### vLLM

1. Instala e inicia vLLM:

```bash
pip install vllm
vllm serve meta-llama/Llama-3.3-70B-Instruct --port 8000
```

2. Configura Claudex:

```toml
[[profiles]]
name = "local-llama"
provider_type = "OpenAICompatible"
base_url = "http://localhost:8000/v1"
api_key = ""
default_model = "meta-llama/Llama-3.3-70B-Instruct"
```

### LM Studio

1. Descarga LM Studio desde [lmstudio.ai](https://lmstudio.ai/)
2. Carga un modelo e inicia el servidor local (puerto por defecto: 1234)

```toml
[[profiles]]
name = "lm-studio"
provider_type = "OpenAICompatible"
base_url = "http://localhost:1234/v1"
api_key = "lm-studio"
default_model = "local-model"
```

## Verificar tu configuracion

Despues de configurar cualquier proveedor, verifica la conectividad:

```bash
# Probar un perfil especifico
claudex profile test <profile-name>

# Probar todos los perfiles
claudex profile test all

# Listar todos los perfiles configurados
claudex profile list
```
