Esta guia cubre las instrucciones detalladas de configuracion para cada proveedor compatible con Claudex, incluyendo proveedores de clave de API, suscripciones OAuth y modelos locales.

## Proveedores con Clave de API

### Anthropic

1. Registrate en [console.anthropic.com](https://console.anthropic.com/)
2. Navega a **API Keys** y crea una nueva clave (formato: `sk-ant-*`)

```toml
[[profiles]]
name = "anthropic"
provider_type = "DirectAnthropic"
base_url = "https://api.anthropic.com"
api_key = "sk-ant-..."
default_model = "claude-sonnet-4-20250514"
```

```bash
claudex profile test anthropic
```

### OpenRouter

1. Registrate en [openrouter.ai](https://openrouter.ai/)
2. Obtener tu clave de API en [openrouter.ai/keys](https://openrouter.ai/keys)

```toml
[[profiles]]
name = "openrouter"
provider_type = "OpenAICompatible"
base_url = "https://openrouter.ai/api/v1"
api_key = "sk-or-..."

```

```bash
claudex profile test openrouter
```

<Aside type="tip">
  OpenRouter es compatible con cientos de modelos de diferentes proveedores. Usa el prefijo `anthropic/` para los modelos Claude.
</Aside>

## Suscripciones OAuth

La autenticacion OAuth te permite usar suscripciones existentes de proveedores (ChatGPT Plus, Claude Max, etc.) en lugar de claves de API separadas.

### Claude Max / Pro

Usa tu suscripcion existente de Claude a traves de la sesion OAuth nativa de Claude Code.

**Requisitos previos**: Claude Code instalado e iniciada la sesion (el comando `claude` funciona normalmente)

```toml
[[profiles]]
name = "claude-sub"
provider_type = "DirectAnthropic"
base_url = "https://api.anthropic.com"
default_model = "claude-sonnet-4-20250514"
auth_type = "oauth"
oauth_provider = "claude"

[profiles.models]
haiku = "claude-haiku-4-20250514"
sonnet = "claude-sonnet-4-20250514"
opus = "claude-opus-4-20250514"
```

<Aside type="note">
  Los perfiles OAuth de Claude usan un modo especial de autenticacion gateway. Claudex establece `ANTHROPIC_AUTH_TOKEN` en lugar de `ANTHROPIC_API_KEY` para evitar conflictos con el inicio de sesion por suscripcion propio de Claude Code. El proxy se omite para los perfiles de suscripcion de Claude.
</Aside>

No se necesita un paso separado de `claudex auth login`. Claudex lee la sesion existente desde `~/.claude`.

### ChatGPT Plus / Codex CLI

Usa tu suscripcion de ChatGPT Plus o Pro mediante el token del Codex CLI.

**Requisitos previos**: Instala el [Codex CLI](https://github.com/openai/codex) y autenticate:

```bash
# Instalar Codex CLI
npm install -g @openai/codex

# Autenticar (abre el navegador)
codex auth
```

Luego configura Claudex:

```toml
[[profiles]]
name = "codex-sub"
provider_type = "OpenAIResponses"
base_url = "https://chatgpt.com/backend-api/codex"
default_model = "gpt-4o"
auth_type = "oauth"
oauth_provider = "openai"

[profiles.models]
haiku = "gpt-4o-mini"
sonnet = "gpt-4o"
opus = "o1-pro"
```

```bash
# Leer token desde Codex CLI
claudex auth login openai --profile codex-sub

# Verificar
claudex auth status

# Ejecutar
claudex run codex-sub
```

<Aside type="tip">
  El proveedor OAuth `openai` usa `OpenAIResponses` como tipo de proveedor. Esto se comunica a traves de la API OpenAI Responses (endpoint `/responses`) en lugar de la API Chat Completions (`/chat/completions`).
</Aside>

### Google Gemini

Usa tu cuenta de Google mediante el flujo de codigo de dispositivo OAuth.

```toml
[[profiles]]
name = "gemini"
provider_type = "OpenAICompatible"
base_url = "https://generativelanguage.googleapis.com/v1beta/openai"
default_model = "gemini-2.5-pro"
auth_type = "oauth"
oauth_provider = "google"

[profiles.models]
haiku = "gemini-2.0-flash"
sonnet = "gemini-2.5-pro"
opus = "gemini-2.5-pro"
```

```bash
# Iniciar el flujo de codigo de dispositivo (abre el navegador para iniciar sesion en Google)
claudex auth login google --profile gemini

# Verificar
claudex auth status
```

## Modelos Locales

### Ollama

1. Instala Ollama: [ollama.com](https://ollama.com/)
2. Descarga un modelo:

```bash
ollama pull qwen2.5:72b
```

3. Configura Claudex:

```toml
[[profiles]]
name = "local-qwen"
provider_type = "OpenAICompatible"
base_url = "http://localhost:11434/v1"
api_key = ""
default_model = "qwen2.5:72b"
```

```bash
claudex profile test local-qwen
```

<Aside type="note">
  Ollama no requiere una clave de API. Deja `api_key` vacio u omitelo.
</Aside>

### vLLM

1. Instala e inicia vLLM:

```bash
pip install vllm
vllm serve meta-llama/Llama-3.3-70B-Instruct --port 8000
```

2. Configura Claudex:

```toml
[[profiles]]
name = "local-llama"
provider_type = "OpenAICompatible"
base_url = "http://localhost:8000/v1"
api_key = ""
default_model = "meta-llama/Llama-3.3-70B-Instruct"
```

### LM Studio

1. Descarga LM Studio desde [lmstudio.ai](https://lmstudio.ai/)
2. Carga un modelo e inicia el servidor local (puerto predeterminado: 1234)

```toml
[[profiles]]
name = "lm-studio"
provider_type = "OpenAICompatible"
base_url = "http://localhost:1234/v1"
api_key = ""
default_model = "loaded-model-name"
```

## Verificar tu Configuracion

Despues de configurar cualquier proveedor, verifica la conectividad:

```bash
# Probar un perfil especifico
claudex profile test <profile-name>

# Probar todos los perfiles
claudex profile test all

# Listar todos los perfiles configurados
claudex profile list
```
