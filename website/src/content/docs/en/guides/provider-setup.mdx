## API Key Providers

### Anthropic

1. Sign up at [console.anthropic.com](https://console.anthropic.com/)
2. Navigate to **API Keys** and create a new key (format: `sk-ant-*`)

```toml
[[profiles]]
name = "anthropic"
provider_type = "DirectAnthropic"
base_url = "https://api.anthropic.com"
api_key = "sk-ant-..."
default_model = "claude-sonnet-4-20250514"
```

```bash
claudex profile test anthropic
```

## Cloud Platforms

### Azure OpenAI

1. Create an Azure OpenAI resource in the [Azure Portal](https://portal.azure.com/)
2. Deploy a model and note the resource name and deployment name
3. Get your API key from **Keys and Endpoint**

```toml
[[profiles]]
name = "azure-openai"
provider_type = "OpenAICompatible"
base_url = "https://YOUR_RESOURCE.openai.azure.com/openai/deployments/YOUR_DEPLOYMENT"
api_key = "YOUR_AZURE_KEY"
default_model = "gpt-4o"

[profiles.query_params]
api-version = "2024-12-01-preview"
```

```bash
claudex profile test azure-openai
```

<Aside type="note">
  Azure OpenAI is auto-detected by `base_url` containing `openai.azure.com`. Authentication uses the `api-key` header instead of `Authorization: Bearer`. The `api-version` query parameter is required.
</Aside>

### Google Vertex AI

1. Enable the Vertex AI API in your GCP project
2. Generate an access token with `gcloud auth print-access-token`

```toml
[[profiles]]
name = "vertex-ai"
provider_type = "DirectAnthropic"
base_url = "https://us-east5-aiplatform.googleapis.com/v1/projects/YOUR_PROJECT/locations/us-east5/publishers/anthropic/models"
api_key = "YOUR_GCLOUD_TOKEN"
default_model = "claude-sonnet-4@20250514"
```

```bash
claudex profile test vertex-ai
```

<Aside type="caution">
  The `gcloud auth print-access-token` output expires after ~1 hour. For production use, consider a service account with token refresh automation.
</Aside>

### AWS Bedrock (via LiteLLM)

AWS Bedrock is supported through a LiteLLM proxy:

1. Install LiteLLM: `pip install litellm`
2. Start the proxy:

```bash
litellm --model bedrock/anthropic.claude-sonnet-4-20250514-v2:0
```

3. Configure Claudex:

```toml
[[profiles]]
name = "bedrock"
provider_type = "OpenAICompatible"
base_url = "http://localhost:4000/v1"
api_key = "sk-litellm"
default_model = "bedrock/anthropic.claude-sonnet-4-20250514-v2:0"
```

## OAuth Subscriptions

OAuth authentication lets you use existing provider subscriptions (ChatGPT Plus, Claude Max, etc.) instead of separate API keys.

### Claude Max / Pro

Use your existing Claude subscription through Claude Code's native OAuth session.

**Prerequisites**: Claude Code installed and logged in (`claude` command works normally)

```toml
[[profiles]]
name = "claude-max"
provider_type = "DirectAnthropic"
base_url = "https://api.claude.ai"
default_model = "claude-sonnet-4-20250514"
auth_type = "oauth"
oauth_provider = "claude"

[profiles.models]
haiku = "claude-haiku-4-20250514"
sonnet = "claude-sonnet-4-20250514"
opus = "claude-opus-4-20250514"
```

<Aside type="note">
  Claude OAuth profiles use a special gateway auth mode. Claudex skips the proxy and lets Claude Code use its own OAuth directly. No separate `claudex auth login` step is needed.
</Aside>

### ChatGPT Plus / Codex CLI

Use your ChatGPT Plus or Pro subscription via the Codex CLI token.

**Prerequisites**: Install [Codex CLI](https://github.com/openai/codex) and authenticate:

```bash
# Install Codex CLI
npm install -g @openai/codex

# Authenticate (opens browser)
codex auth
```

Then configure Claudex:

```toml
[[profiles]]
name = "codex-sub"
provider_type = "OpenAIResponses"
base_url = "https://chatgpt.com/backend-api/codex"
default_model = "gpt-5.3-codex"
auth_type = "oauth"
oauth_provider = "openai"

[profiles.models]
haiku = "codex-mini-latest"
sonnet = "gpt-5.3-codex"
opus = "gpt-5.3-codex"
```

```bash
# Read token from Codex CLI
claudex auth login openai --profile codex-sub

# Verify
claudex auth status

# Run
claudex run codex-sub
```

<Aside type="tip">
  Supported models on the Codex endpoint: `gpt-5.3-codex`, `gpt-5.1-codex`, `codex-mini-latest`. Note that `gpt-4o` is NOT supported on the `chatgpt.com` Codex endpoint. `strip_params = "auto"` will automatically detect this endpoint and strip `temperature`/`top_p`/`top_k`.
</Aside>

</Aside>

## Local Models

### Ollama

1. Install Ollama: [ollama.com](https://ollama.com/)
2. Pull a model:

```bash
ollama pull qwen2.5:72b
```

3. Configure Claudex:

```toml
[[profiles]]
name = "local-qwen"
provider_type = "OpenAICompatible"
base_url = "http://localhost:11434/v1"
api_key = ""
default_model = "qwen2.5:72b"
```

```bash
claudex profile test local-qwen
```

<Aside type="note">
  Ollama does not require an API key. Leave `api_key` empty or omit it.
</Aside>

### vLLM

1. Install and start vLLM:

```bash
pip install vllm
vllm serve meta-llama/Llama-3.3-70B-Instruct --port 8000
```

2. Configure Claudex:

```toml
[[profiles]]
name = "local-llama"
provider_type = "OpenAICompatible"
base_url = "http://localhost:8000/v1"
api_key = ""
default_model = "meta-llama/Llama-3.3-70B-Instruct"
```

### LM Studio

1. Download LM Studio from [lmstudio.ai](https://lmstudio.ai/)
2. Load a model and start the local server (default port: 1234)

```toml
[[profiles]]
name = "lm-studio"
provider_type = "OpenAICompatible"
base_url = "http://localhost:1234/v1"
api_key = "lm-studio"
default_model = "local-model"
```

## Verifying Your Setup

After configuring any provider, verify connectivity:

```bash
# Test a specific profile
claudex profile test <profile-name>

# Test all profiles
claudex profile test all

# List all configured profiles
claudex profile list
```
