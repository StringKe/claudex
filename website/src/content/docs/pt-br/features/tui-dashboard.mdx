---
title: Dashboard TUI
description: Dashboard em tempo real no terminal para monitoramento e gerenciamento de profiles
---

O dashboard TUI fornece uma interface em tempo real no terminal para monitorar a saude dos profiles, visualizar metricas e iniciar sessoes do Claude Code rapidamente.

## Iniciar

```bash
# Inicializacao explicita
claudex dashboard

# Inicializacao automatica (quando profiles existem e nenhum subcomando e fornecido)
claudex
```

## Atalhos de Teclado

| Tecla | Acao |
|-------|------|
| `j` / `↓` | Mover selecao para baixo |
| `k` / `↑` | Mover selecao para cima |
| `Enter` | Iniciar Claude Code com o profile selecionado |
| `t` | Testar conectividade do profile selecionado |
| `a` | Adicionar novo profile (exibe dica) |
| `d` | Excluir profile selecionado (exibe dica) |
| `p` | Alternar iniciar/parar proxy |
| `/` | Entrar no modo de busca |
| `Esc` | Sair do modo de busca / fechar ajuda |
| `?` | Exibir popup de ajuda |
| `q` / `Ctrl+C` | Sair do dashboard |

## Layout do Dashboard

O dashboard exibe:

- **Lista de profiles** — Todos os profiles configurados com nome, modelo, tipo de provedor e status de saude
- **Indicadores de saude** — Status codificado por cores (verde = saudavel, amarelo = degradado, vermelho = circuit breaker aberto)
- **Painel de metricas** — Contagem de requisicoes por profile, taxa de sucesso, latencia media
- **Painel de logs** — Saida de log em rolagem em tempo real

## Status de Saude

O dashboard executa um verificador de saude em segundo plano que envia pings para todos os profiles habilitados a cada ~30 segundos. O status de saude e refletido em tempo real:

| Cor | Significado |
|-----|-------------|
| Verde | Provedor esta respondendo normalmente |
| Amarelo | Falhas recentes detectadas |
| Vermelho | Circuit breaker esta aberto |
| Cinza | Profile esta desabilitado |
