## Uso Global

```
claudex [OPTIONS] [COMMAND]
```

| Opcao | Descricao |
|-------|-----------|
| `--config <PATH>` | Substituir caminho do arquivo de configuracao |
| `--version` | Exibir versao |
| `--help` | Exibir ajuda |

Quando executado sem um comando, o Claudex inicia o dashboard TUI se profiles estiverem configurados, ou exibe uma mensagem de boas-vindas se nenhum profile existir.

## Comandos

### `run`

Executar o Claude Code com um profile de provedor especifico.

```bash
claudex run <PROFILE> [PROMPT] [OPTIONS] [-- ARGS...]
```

| Argumento / Opcao | Descricao |
|-------------------|-----------|
| `PROFILE` | Nome do profile, ou `auto` para roteamento inteligente |
| `PROMPT` | Prompt opcional para execucao nao-interativa (one-shot) |
| `-m, --model MODEL` | Substituir o modelo padrao para esta sessao |
| `--hyperlinks` | Forcar ativacao de hyperlinks OSC 8 no terminal |
| `--print` | Imprimir a resposta e sair (modo nao-interativo) |
| `--dangerously-skip-permissions` | Pular todas as solicitacoes de permissao (use com `--print` para CI/CD) |
| `ARGS...` | Argumentos adicionais passados para `claude` (apos `--`) |

```bash
# Usar Grok (interativo)
claudex run grok

# Usar Grok com um modelo especifico
claudex run grok -m grok-3-mini-beta

# Roteamento inteligente
claudex run auto

# Forcar hyperlinks no terminal
claudex run grok --hyperlinks

# Execucao one-shot nao-interativa
claudex run grok "Explain this codebase" --print

# Uso em pipeline CI/CD (sem prompts, sem permissoes)
claudex run grok "Fix lint errors" --print --dangerously-skip-permissions

# Passar argumentos extras para claude
claudex run grok -- --verbose
```

### `profile`

Gerenciar profiles de provedores.

#### `profile list`

```bash
claudex profile list
```

Listar todos os profiles configurados com status, tipo de provedor e modelo.

#### `profile add`

```bash
claudex profile add
```

Assistente interativo para adicionar um novo profile. Guia pela selecao do provedor, entrada da chave API (com armazenamento opcional no keyring), selecao de modelo e teste de conectividade.

#### `profile show`

```bash
claudex profile show <NAME>
```

Exibir detalhes de um profile especifico incluindo todos os campos de configuracao.

#### `profile remove`

```bash
claudex profile remove <NAME>
```

Remover um profile da configuracao.

#### `profile test`

```bash
claudex profile test <NAME|all>
```

Testar conectividade para um profile especifico ou todos os profiles. Envia uma requisicao leve ao endpoint `/models` do provedor.

### `auth`

Gerenciar autenticacao OAuth por assinatura para profiles de provedores.

#### `auth login`

```bash
claudex auth login <PROVIDER> [OPTIONS]
```

| Opcao | Descricao |
|-------|-----------|
| `--profile <NAME>` | Associar a um profile especifico |
| `--enterprise-url <URL>` | URL de servidor enterprise personalizado (para GitLab auto-hospedado, etc.) |
| `--headless` | Usar fluxo headless (device code), pular lancamento do navegador |

Fazer login em um provedor usando OAuth. Provedores suportados: `claude`, `openai`, `google`, `qwen`, `kimi`, `github`, `gitlab`.

```bash
# Fazer login na OpenAI (le do Codex CLI)
claudex auth login openai --profile codex-sub

# Fazer login no Claude (le de ~/.claude)
claudex auth login claude

# Fazer login no Google
claudex auth login google --profile gemini-sub

# Fazer login no GitLab com URL enterprise
claudex auth login gitlab --enterprise-url https://gitlab.mycompany.com

# Modo headless (sem navegador, apenas device code)
claudex auth login github --headless
```

#### `auth status`

```bash
claudex auth status [PROVIDER]
```

Exibir o status de autenticacao atual. Sem argumento de provedor, exibe o status de todos os profiles OAuth configurados.

```bash
# Verificar todos os profiles OAuth
claudex auth status

# Verificar um provedor especifico
claudex auth status openai
```

#### `auth refresh`

```bash
claudex auth refresh <PROVIDER>
```

Atualizar manualmente o token OAuth de um provedor. Os tokens tambem sao atualizados automaticamente pelo proxy quando expiram.

#### `auth logout`

```bash
claudex auth logout <PROVIDER>
```

Remover tokens OAuth armazenados de um provedor do keyring do sistema.

### `proxy`

Gerenciar o servidor proxy de traducao.

#### `proxy start`

```bash
claudex proxy start [OPTIONS]
```

| Flag | Descricao |
|------|-----------|
| `-p, --port PORT` | Substituir porta do proxy (padrao: da configuracao) |
| `-d, --daemon` | Executar como daemon em segundo plano |

#### `proxy stop`

```bash
claudex proxy stop
```

Parar o daemon do proxy (envia SIGTERM via arquivo PID).

#### `proxy status`

```bash
claudex proxy status
```

Exibir se o proxy esta em execucao e em qual porta.

### `dashboard`

```bash
claudex dashboard
```

Iniciar o dashboard TUI para monitoramento em tempo real e lancamento rapido.

### `config`

Gerenciar arquivos de configuracao.

#### `config show`

```bash
claudex config show
```

Exibir o caminho do arquivo de configuracao carregado, ordem de busca e valores de configuracao atuais.

#### `config path`

```bash
claudex config path
```

Imprimir apenas o caminho resolvido do arquivo de configuracao.

#### `config init`

```bash
claudex config init
```

Criar um novo arquivo de configuracao no diretorio atual a partir do template embutido.

#### `config recreate`

```bash
claudex config recreate
```

Recriar o arquivo de configuracao a partir do `config.example.toml`, sobrescrevendo o config existente.

#### `config edit`

```bash
claudex config edit
```

Abrir o arquivo de configuracao no seu `$EDITOR` (ou `$VISUAL`).

#### `config validate`

```bash
claudex config validate
```

Validar sintaxe do config, referencias de profiles (backup_providers, regras do router) e configuracoes de provedores OAuth. Reporta erros e avisos.

#### `config get`

```bash
claudex config get <KEY>
```

Obter um valor especifico de configuracao por caminho de chave (ex: `proxy_port`, `profiles.0.name`).

#### `config set`

```bash
claudex config set <KEY> <VALUE>
```

Definir um valor especifico de configuracao por caminho de chave.

#### `config export`

```bash
claudex config export
```

Exportar a configuracao mesclada atual para stdout no formato TOML.

### `sets`

Gerenciar conjuntos de configuracao (pacotes instalaveis de regras, skills e servidores MCP).

#### `sets add`

```bash
claudex sets add <PATH>
```

Instalar um conjunto de configuracao a partir de um diretorio local contendo um `manifest.json` (ou `manifest.yaml`).

#### `sets remove`

```bash
claudex sets remove <NAME>
```

Remover um conjunto de configuracao instalado por nome.

#### `sets list`

```bash
claudex sets list
```

Listar todos os conjuntos de configuracao instalados com versao, descricao e resumo de componentes.

#### `sets update`

```bash
claudex sets update <NAME> <PATH>
```

Atualizar um conjunto de configuracao existente a partir de um novo caminho de manifesto.

#### `sets show`

```bash
claudex sets show <NAME>
```

Exibir informacoes detalhadas sobre um conjunto de configuracao instalado, incluindo todos os componentes.

### `update`

```bash
claudex update [OPTIONS]
```

| Flag | Descricao |
|------|-----------|
| `--check` | Apenas verificar atualizacoes, sem instalar |

Sem `--check`, baixa e instala a versao mais recente do GitHub Releases.
