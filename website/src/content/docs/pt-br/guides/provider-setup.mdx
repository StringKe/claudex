Este guia cobre instrucoes detalhadas de configuracao para cada provedor suportado pelo Claudex, incluindo provedores por chave de API, assinaturas OAuth e modelos locais.

## Provedores por Chave de API

### Anthropic

1. Cadastre-se em [console.anthropic.com](https://console.anthropic.com/)
2. Navegue ate **API Keys** e crie uma nova chave (formato: `sk-ant-*`)

```toml
[[profiles]]
name = "anthropic"
provider_type = "DirectAnthropic"
base_url = "https://api.anthropic.com"
api_key = "sk-ant-..."
default_model = "claude-sonnet-4-20250514"
```

```bash
claudex profile test anthropic
```

### MiniMax

2. Obtenha sua chave de API em [openrouter.ai/keys](https://openrouter.ai/keys)


## Assinaturas OAuth

A autenticacao OAuth permite usar assinaturas de provedores existentes (ChatGPT Plus, Claude Max, etc.) em vez de chaves de API separadas.

### Claude Max / Pro

Use sua assinatura Claude existente atraves da sessao OAuth nativa do Claude Code.

**Pre-requisitos**: Claude Code instalado e autenticado (o comando `claude` funciona normalmente)

```toml
[[profiles]]
name = "claude-sub"
provider_type = "DirectAnthropic"
base_url = "https://api.anthropic.com"
default_model = "claude-sonnet-4-20250514"
auth_type = "oauth"
oauth_provider = "claude"

[profiles.models]
haiku = "claude-haiku-4-20250514"
sonnet = "claude-sonnet-4-20250514"
opus = "claude-opus-4-20250514"
```

<Aside type="note">
  Perfis OAuth do Claude usam um modo especial de autenticacao gateway. O Claudex define `ANTHROPIC_AUTH_TOKEN` em vez de `ANTHROPIC_API_KEY` para evitar conflitos com o proprio login de assinatura do Claude Code. O proxy e ignorado para perfis de assinatura Claude.
</Aside>

Nenhuma etapa separada de `claudex auth login` e necessaria. O Claudex le a sessao existente de `~/.claude`.

### ChatGPT Plus / Codex CLI

Use sua assinatura ChatGPT Plus ou Pro via token do Codex CLI.

**Pre-requisitos**: Instale o [Codex CLI](https://github.com/openai/codex) e autentique-se:

```bash
# Instalar Codex CLI
npm install -g @openai/codex

# Autenticar (abre o navegador)
codex auth
```

Em seguida, configure o Claudex:

```toml
[[profiles]]
name = "codex-sub"
provider_type = "OpenAIResponses"
base_url = "https://chatgpt.com/backend-api/codex"
default_model = "gpt-4o"
auth_type = "oauth"
oauth_provider = "openai"

[profiles.models]
haiku = "gpt-4o-mini"
sonnet = "gpt-4o"
opus = "o1-pro"
```

```bash
# Ler token do Codex CLI
claudex auth login openai --profile codex-sub

# Verificar
claudex auth status

# Executar
claudex run codex-sub
```

<Aside type="tip">
  O provedor OAuth `openai` usa `OpenAIResponses` como tipo de provedor. Isso se comunica atraves da API OpenAI Responses (endpoint `/responses`) em vez da API Chat Completions (`/chat/completions`).
</Aside>

### Google Gemini

Use sua conta Google via fluxo OAuth de codigo de dispositivo.

```toml
[[profiles]]
name = "gemini"
provider_type = "OpenAICompatible"
base_url = "https://generativelanguage.googleapis.com/v1beta/openai"
default_model = "gemini-2.5-pro"
auth_type = "oauth"
oauth_provider = "google"

[profiles.models]
haiku = "gemini-2.0-flash"
sonnet = "gemini-2.5-pro"
opus = "gemini-2.5-pro"
```

```bash
# Iniciar fluxo de codigo de dispositivo (abre o navegador para login Google)
claudex auth login google --profile gemini

# Verificar
claudex auth status
```

## Modelos Locais

### Ollama

1. Instale o Ollama: [ollama.com](https://ollama.com/)
2. Baixe um modelo:

```bash
ollama pull qwen2.5:72b
```

3. Configure o Claudex:

```toml
[[profiles]]
name = "local-qwen"
provider_type = "OpenAICompatible"
base_url = "http://localhost:11434/v1"
api_key = ""
default_model = "qwen2.5:72b"
```

```bash
claudex profile test local-qwen
```

<Aside type="note">
  O Ollama nao requer uma chave de API. Deixe `api_key` vazio ou omita-o.
</Aside>

### vLLM

1. Instale e inicie o vLLM:

```bash
pip install vllm
vllm serve meta-llama/Llama-3.3-70B-Instruct --port 8000
```

2. Configure o Claudex:

```toml
[[profiles]]
name = "local-llama"
provider_type = "OpenAICompatible"
base_url = "http://localhost:8000/v1"
api_key = ""
default_model = "meta-llama/Llama-3.3-70B-Instruct"
```

### LM Studio

1. Baixe o LM Studio em [lmstudio.ai](https://lmstudio.ai/)
2. Carregue um modelo e inicie o servidor local (porta padrao: 1234)

```toml
[[profiles]]
name = "lm-studio"
provider_type = "OpenAICompatible"
base_url = "http://localhost:1234/v1"
api_key = ""
default_model = "loaded-model-name"
```

## Verificando sua Configuracao

Apos configurar qualquer provedor, verifique a conectividade:

```bash
# Testar um perfil especifico
claudex profile test <nome-do-perfil>

# Testar todos os perfis
claudex profile test all

# Listar todos os perfis configurados
claudex profile list
```
