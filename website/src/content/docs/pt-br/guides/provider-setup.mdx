Este guia cobre instrucoes detalhadas de configuracao para todos os provedores suportados pelo Claudex, incluindo provedores com chave API, assinaturas OAuth, plataformas cloud e modelos locais.

## Provedores com Chave API

### Anthropic

1. Cadastre-se em [console.anthropic.com](https://console.anthropic.com/)
2. Navegue ate **API Keys** e crie uma nova chave (formato: `sk-ant-*`)

```toml
[[profiles]]
name = "anthropic"
provider_type = "DirectAnthropic"
base_url = "https://api.anthropic.com"
api_key = "sk-ant-..."
default_model = "claude-sonnet-4-20250514"
```

```bash
claudex profile test anthropic
```

## Plataformas Cloud

### Azure OpenAI

1. Crie um recurso Azure OpenAI no [Portal Azure](https://portal.azure.com/)
2. Faca deploy de um modelo e anote o nome do recurso e do deployment
3. Obtenha sua chave API em **Keys and Endpoint**

```toml
[[profiles]]
name = "azure-openai"
provider_type = "OpenAICompatible"
base_url = "https://YOUR_RESOURCE.openai.azure.com/openai/deployments/YOUR_DEPLOYMENT"
api_key = "YOUR_AZURE_KEY"
default_model = "gpt-4o"

[profiles.query_params]
api-version = "2024-12-01-preview"
```

```bash
claudex profile test azure-openai
```

<Aside type="note">
  O Azure OpenAI e detectado automaticamente quando a `base_url` contem `openai.azure.com`. A autenticacao usa o header `api-key` em vez de `Authorization: Bearer`. O parametro de query `api-version` e obrigatorio.
</Aside>

### Google Vertex AI

1. Habilite a API Vertex AI no seu projeto GCP
2. Gere um token de acesso com `gcloud auth print-access-token`

```toml
[[profiles]]
name = "vertex-ai"
provider_type = "DirectAnthropic"
base_url = "https://us-east5-aiplatform.googleapis.com/v1/projects/YOUR_PROJECT/locations/us-east5/publishers/anthropic/models"
api_key = "YOUR_GCLOUD_TOKEN"
default_model = "claude-sonnet-4@20250514"
```

```bash
claudex profile test vertex-ai
```

<Aside type="caution">
  A saida do `gcloud auth print-access-token` expira apos ~1 hora. Para uso em producao, considere uma conta de servico com automacao de refresh de token.
</Aside>

### AWS Bedrock (via LiteLLM)

O AWS Bedrock e suportado atraves de um proxy LiteLLM:

1. Instale o LiteLLM: `pip install litellm`
2. Inicie o proxy:

```bash
litellm --model bedrock/anthropic.claude-sonnet-4-20250514-v2:0
```

3. Configure o Claudex:

```toml
[[profiles]]
name = "bedrock"
provider_type = "OpenAICompatible"
base_url = "http://localhost:4000/v1"
api_key = "sk-litellm"
default_model = "bedrock/anthropic.claude-sonnet-4-20250514-v2:0"
```

## Assinaturas OAuth

A autenticacao OAuth permite usar assinaturas existentes de provedores (ChatGPT Plus, Claude Max, etc.) em vez de chaves API separadas.

### Claude Max / Pro

Use sua assinatura Claude existente atraves da sessao OAuth nativa do Claude Code.

**Pre-requisitos**: Claude Code instalado e logado (o comando `claude` funciona normalmente)

```toml
[[profiles]]
name = "claude-max"
provider_type = "DirectAnthropic"
base_url = "https://api.claude.ai"
default_model = "claude-sonnet-4-20250514"
auth_type = "oauth"
oauth_provider = "claude"

[profiles.models]
haiku = "claude-haiku-4-20250514"
sonnet = "claude-sonnet-4-20250514"
opus = "claude-opus-4-20250514"
```

<Aside type="note">
  Profiles OAuth Claude usam um modo especial de gateway auth. O Claudex pula o proxy e deixa o Claude Code usar seu proprio OAuth diretamente. Nenhum passo separado de `claudex auth login` e necessario.
</Aside>

### ChatGPT Plus / Codex CLI

Use sua assinatura ChatGPT Plus ou Pro via token do Codex CLI.

**Pre-requisitos**: Instale o [Codex CLI](https://github.com/openai/codex) e autentique:

```bash
# Instalar Codex CLI
npm install -g @openai/codex

# Autenticar (abre o navegador)
codex auth
```

Em seguida, configure o Claudex:

```toml
[[profiles]]
name = "codex-sub"
provider_type = "OpenAIResponses"
base_url = "https://chatgpt.com/backend-api/codex"
default_model = "gpt-5.3-codex"
auth_type = "oauth"
oauth_provider = "openai"

[profiles.models]
haiku = "codex-mini-latest"
sonnet = "gpt-5.3-codex"
opus = "gpt-5.3-codex"
```

```bash
# Ler token do Codex CLI
claudex auth login openai --profile codex-sub

# Verificar
claudex auth status

# Executar
claudex run codex-sub
```

<Aside type="tip">
  Modelos suportados no endpoint Codex: `gpt-5.3-codex`, `gpt-5.1-codex`, `codex-mini-latest`. Note que `gpt-4o` NAO e suportado no endpoint Codex do `chatgpt.com`. `strip_params = "auto"` detectara automaticamente este endpoint e removera `temperature`/`top_p`/`top_k`.
</Aside>

```bash

claudex auth status
```

<Aside type="note">
  O GitLab Duo requer uma variavel de ambiente `GITLAB_TOKEN` com um Personal Access Token que tenha o escopo `ai_features`. Para instancias GitLab auto-hospedadas, atualize a `base_url` conforme necessario.
</Aside>

## Modelos Locais

### Ollama

1. Instale o Ollama: [ollama.com](https://ollama.com/)
2. Baixe um modelo:

```bash
ollama pull qwen2.5:72b
```

3. Configure o Claudex:

```toml
[[profiles]]
name = "local-qwen"
provider_type = "OpenAICompatible"
base_url = "http://localhost:11434/v1"
api_key = ""
default_model = "qwen2.5:72b"
```

```bash
claudex profile test local-qwen
```

<Aside type="note">
  O Ollama nao requer chave API. Deixe `api_key` vazio ou omita.
</Aside>

### vLLM

1. Instale e inicie o vLLM:

```bash
pip install vllm
vllm serve meta-llama/Llama-3.3-70B-Instruct --port 8000
```

2. Configure o Claudex:

```toml
[[profiles]]
name = "local-llama"
provider_type = "OpenAICompatible"
base_url = "http://localhost:8000/v1"
api_key = ""
default_model = "meta-llama/Llama-3.3-70B-Instruct"
```

### LM Studio

1. Baixe o LM Studio em [lmstudio.ai](https://lmstudio.ai/)
2. Carregue um modelo e inicie o servidor local (porta padrao: 1234)

```toml
[[profiles]]
name = "lm-studio"
provider_type = "OpenAICompatible"
base_url = "http://localhost:1234/v1"
api_key = "lm-studio"
default_model = "local-model"
```

## Verificando sua Configuracao

Apos configurar qualquer provedor, verifique a conectividade:

```bash
# Testar um profile especifico
claudex profile test <profile-name>

# Testar todos os profiles
claudex profile test all

# Listar todos os profiles configurados
claudex profile list
```
