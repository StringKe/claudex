## Общее использование

```
claudex [OPTIONS] [COMMAND]
```

| Опция | Описание |
|-------|---------|
| `--config <PATH>` | Переопределить путь к файлу конфигурации |
| `--version` | Вывести версию |
| `--help` | Вывести справку |

При запуске без команды Claudex запускает TUI-панель мониторинга, если профили настроены, или показывает приветственное сообщение, если профили отсутствуют.

## Команды

### `run`

Запустить Claude Code с конкретным профилем провайдера.

```bash
claudex run <PROFILE> [PROMPT] [OPTIONS] [-- ARGS...]
```

| Аргумент / Опция | Описание |
|----------|---------|
| `PROFILE` | Имя профиля или `auto` для умной маршрутизации |
| `PROMPT` | Опциональный промпт для неинтерактивного (одиночного) выполнения |
| `-m, --model MODEL` | Переопределить модель по умолчанию для этой сессии |
| `--hyperlinks` | Принудительно включить терминальные гиперссылки OSC 8 |
| `--print` | Вывести ответ и выйти (неинтерактивный режим) |
| `--dangerously-skip-permissions` | Пропустить все запросы разрешений (использовать с `--print` для CI/CD) |
| `ARGS...` | Дополнительные аргументы, передаваемые в `claude` (после `--`) |

```bash
# Использовать Grok (интерактивно)
claudex run grok

# Использовать Grok с конкретной моделью
claudex run grok -m grok-3-mini-beta

# Умная маршрутизация
claudex run auto

# Принудительно включить терминальные гиперссылки
claudex run grok --hyperlinks

# Неинтерактивное одиночное выполнение
claudex run grok "Explain this codebase" --print

# Использование в CI/CD-конвейере (без запросов, без разрешений)
claudex run grok "Fix lint errors" --print --dangerously-skip-permissions

# Передать дополнительные аргументы в claude
claudex run grok -- --verbose
```

### `profile`

Управление профилями провайдеров.

#### `profile list`

```bash
claudex profile list
```

Список всех настроенных профилей с их статусом, типом провайдера и моделью.

#### `profile add`

```bash
claudex profile add
```

Интерактивный мастер добавления нового профиля. Проводит через выбор провайдера, ввод API-ключа (с опциональным сохранением в связке ключей), выбор модели и тестирование подключения.

#### `profile show`

```bash
claudex profile show <NAME>
```

Показать подробности конкретного профиля, включая все поля конфигурации.

#### `profile remove`

```bash
claudex profile remove <NAME>
```

Удалить профиль из конфигурации.

#### `profile test`

```bash
claudex profile test <NAME|all>
```

Тестировать подключение для конкретного профиля или всех профилей. Отправляет лёгкий запрос к endpoint `/models` провайдера.

### `auth`

Управление OAuth-аутентификацией по подписке для профилей провайдеров.

#### `auth login`

```bash
claudex auth login <PROVIDER> [OPTIONS]
```

| Опция | Описание |
|-------|---------|
| `--profile <NAME>` | Привязать к конкретному профилю |
| `--enterprise-url <URL>` | URL корпоративного сервера (для self-hosted GitLab и т.д.) |
| `--headless` | Использовать headless (device code) flow, пропустить запуск браузера |

Вход к провайдеру через OAuth. Поддерживаемые провайдеры: `claude`, `openai`, `google`, `qwen`, `kimi`, `github`, `gitlab`.

```bash
# Войти в OpenAI (читает из Codex CLI)
claudex auth login openai --profile codex-sub

# Войти в Claude (читает из ~/.claude)
claudex auth login claude

# Войти в Google
claudex auth login google --profile gemini-sub

# Войти в GitLab с enterprise URL
claudex auth login gitlab --enterprise-url https://gitlab.mycompany.com

# Headless-режим (без браузера, только device code)
claudex auth login github --headless
```

#### `auth status`

```bash
claudex auth status [PROVIDER]
```

Показать текущий статус аутентификации. Без аргумента провайдера показывает статус всех настроенных OAuth-профилей.

```bash
# Проверить все OAuth-профили
claudex auth status

# Проверить конкретного провайдера
claudex auth status openai
```

#### `auth refresh`

```bash
claudex auth refresh <PROVIDER>
```

Вручную обновить OAuth-токен для провайдера. Токены также обновляются автоматически прокси при истечении срока.

#### `auth logout`

```bash
claudex auth logout <PROVIDER>
```

Удалить сохранённые OAuth-токены для провайдера из системной связки ключей.

### `proxy`

Управление сервером прокси-переводчика.

#### `proxy start`

```bash
claudex proxy start [OPTIONS]
```

| Флаг | Описание |
|------|---------|
| `-p, --port PORT` | Переопределить порт прокси (по умолчанию: из конфигурации) |
| `-d, --daemon` | Запустить как фоновый демон |

#### `proxy stop`

```bash
claudex proxy stop
```

Остановить демон прокси (отправляет SIGTERM через PID-файл).

#### `proxy status`

```bash
claudex proxy status
```

Показать, запущен ли прокси и на каком порту.

### `dashboard`

```bash
claudex dashboard
```

Запустить TUI-панель мониторинга для наблюдения в реальном времени и быстрого запуска.

### `config`

Управление конфигурационными файлами.

#### `config show`

```bash
claudex config show
```

Отобразить путь к загруженному файлу конфигурации, порядок поиска и текущие значения.

#### `config path`

```bash
claudex config path
```

Вывести только разрешённый путь к файлу конфигурации.

#### `config init`

```bash
claudex config init
```

Создать новый файл конфигурации в текущем каталоге из встроенного шаблона.

#### `config recreate`

```bash
claudex config recreate
```

Пересоздать файл конфигурации из `config.example.toml`, перезаписав существующую конфигурацию.

#### `config edit`

```bash
claudex config edit
```

Открыть файл конфигурации в вашем `$EDITOR` (или `$VISUAL`).

#### `config validate`

```bash
claudex config validate
```

Проверить синтаксис конфигурации, ссылки профилей (backup_providers, правила маршрутизации) и настройки OAuth-провайдеров. Выводит ошибки и предупреждения.

#### `config get`

```bash
claudex config get <KEY>
```

Получить конкретное значение конфигурации по ключу (например, `proxy_port`, `profiles.0.name`).

#### `config set`

```bash
claudex config set <KEY> <VALUE>
```

Установить конкретное значение конфигурации по ключу.

#### `config export`

```bash
claudex config export
```

Экспортировать текущую объединённую конфигурацию в stdout в формате TOML.

### `sets`

Управление конфигурационными наборами (устанавливаемые пакеты правил, навыков и MCP-серверов).

#### `sets add`

```bash
claudex sets add <PATH>
```

Установить конфигурационный набор из локального каталога, содержащего `manifest.json` (или `manifest.yaml`).

#### `sets remove`

```bash
claudex sets remove <NAME>
```

Удалить установленный конфигурационный набор по имени.

#### `sets list`

```bash
claudex sets list
```

Список всех установленных конфигурационных наборов с их версией, описанием и сводкой компонентов.

#### `sets update`

```bash
claudex sets update <NAME> <PATH>
```

Обновить существующий конфигурационный набор из нового пути к манифесту.

#### `sets show`

```bash
claudex sets show <NAME>
```

Показать подробную информацию об установленном конфигурационном наборе, включая все компоненты.

### `update`

```bash
claudex update [OPTIONS]
```

| Флаг | Описание |
|------|---------|
| `--check` | Только проверить наличие обновлений, не устанавливать |

Без `--check` загружает и устанавливает последнюю версию из GitHub Releases.
