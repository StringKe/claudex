Это руководство содержит подробные инструкции по настройке всех провайдеров, поддерживаемых Claudex, включая провайдеров с API-ключами, OAuth-подписки, облачные платформы и локальные модели.

## Провайдеры с API-ключами

### Anthropic

1. Зарегистрируйтесь на [console.anthropic.com](https://console.anthropic.com/)
2. Перейдите в **API Keys** и создайте новый ключ (формат: `sk-ant-*`)

```toml
[[profiles]]
name = "anthropic"
provider_type = "DirectAnthropic"
base_url = "https://api.anthropic.com"
api_key = "sk-ant-..."
default_model = "claude-sonnet-4-20250514"
```

```bash
claudex profile test anthropic
```

## Облачные платформы

### Azure OpenAI

1. Создайте ресурс Azure OpenAI в [Azure Portal](https://portal.azure.com/)
2. Разверните модель и запишите имя ресурса и имя развёртывания
3. Получите API-ключ в разделе **Keys and Endpoint**

```toml
[[profiles]]
name = "azure-openai"
provider_type = "OpenAICompatible"
base_url = "https://YOUR_RESOURCE.openai.azure.com/openai/deployments/YOUR_DEPLOYMENT"
api_key = "YOUR_AZURE_KEY"
default_model = "gpt-4o"

[profiles.query_params]
api-version = "2024-12-01-preview"
```

```bash
claudex profile test azure-openai
```

<Aside type="note">
  Azure OpenAI автоматически определяется по `base_url`, содержащему `openai.azure.com`. Аутентификация использует заголовок `api-key` вместо `Authorization: Bearer`. Query-параметр `api-version` обязателен.
</Aside>

### Google Vertex AI

1. Включите API Vertex AI в вашем GCP-проекте
2. Сгенерируйте токен доступа с помощью `gcloud auth print-access-token`

```toml
[[profiles]]
name = "vertex-ai"
provider_type = "DirectAnthropic"
base_url = "https://us-east5-aiplatform.googleapis.com/v1/projects/YOUR_PROJECT/locations/us-east5/publishers/anthropic/models"
api_key = "YOUR_GCLOUD_TOKEN"
default_model = "claude-sonnet-4@20250514"
```

```bash
claudex profile test vertex-ai
```

<Aside type="caution">
  Вывод `gcloud auth print-access-token` истекает примерно через 1 час. Для промышленного использования рассмотрите сервисный аккаунт с автоматическим обновлением токена.
</Aside>

### AWS Bedrock (через LiteLLM)

AWS Bedrock поддерживается через прокси LiteLLM:

1. Установите LiteLLM: `pip install litellm`
2. Запустите прокси:

```bash
litellm --model bedrock/anthropic.claude-sonnet-4-20250514-v2:0
```

3. Настройте Claudex:

```toml
[[profiles]]
name = "bedrock"
provider_type = "OpenAICompatible"
base_url = "http://localhost:4000/v1"
api_key = "sk-litellm"
default_model = "bedrock/anthropic.claude-sonnet-4-20250514-v2:0"
```

## OAuth-подписки

OAuth-аутентификация позволяет использовать существующие подписки провайдеров (ChatGPT Plus, Claude Max и т.д.) вместо отдельных API-ключей.

### Claude Max / Pro

Используйте существующую подписку Claude через нативную OAuth-сессию Claude Code.

**Предварительные условия**: Claude Code установлен и выполнен вход (команда `claude` работает нормально)

```toml
[[profiles]]
name = "claude-max"
provider_type = "DirectAnthropic"
base_url = "https://api.claude.ai"
default_model = "claude-sonnet-4-20250514"
auth_type = "oauth"
oauth_provider = "claude"

[profiles.models]
haiku = "claude-haiku-4-20250514"
sonnet = "claude-sonnet-4-20250514"
opus = "claude-opus-4-20250514"
```

<Aside type="note">
  OAuth-профили Claude используют специальный режим gateway auth. Claudex пропускает прокси и позволяет Claude Code использовать собственный OAuth напрямую. Отдельный шаг `claudex auth login` не требуется.
</Aside>

### ChatGPT Plus / Codex CLI

Используйте подписку ChatGPT Plus или Pro через токен Codex CLI.

**Предварительные условия**: Установите [Codex CLI](https://github.com/openai/codex) и выполните аутентификацию:

```bash
# Установить Codex CLI
npm install -g @openai/codex

# Аутентифицироваться (открывает браузер)
codex auth
```

Затем настройте Claudex:

```toml
[[profiles]]
name = "codex-sub"
provider_type = "OpenAIResponses"
base_url = "https://chatgpt.com/backend-api/codex"
default_model = "gpt-5.3-codex"
auth_type = "oauth"
oauth_provider = "openai"

[profiles.models]
haiku = "codex-mini-latest"
sonnet = "gpt-5.3-codex"
opus = "gpt-5.3-codex"
```

```bash
# Считать токен из Codex CLI
claudex auth login openai --profile codex-sub

# Проверить
claudex auth status

# Запустить
claudex run codex-sub
```

<Aside type="tip">
  Поддерживаемые модели на Codex endpoint: `gpt-5.3-codex`, `gpt-5.1-codex`, `codex-mini-latest`. Модель `gpt-4o` НЕ поддерживается на endpoint `chatgpt.com` Codex. `strip_params = "auto"` автоматически обнаруживает этот endpoint и удаляет `temperature`/`top_p`/`top_k`.
</Aside>

### GitLab Duo

Используйте подписку GitLab Duo через Personal Access Token.

```toml
[[profiles]]
name = "gitlab-duo"
provider_type = "OpenAICompatible"
base_url = "https://gitlab.com/api/v4/ai/llm/proxy"
default_model = "claude-sonnet-4-20250514"

oauth_provider = "gitlab"
```

```bash
# Установите токен GitLab как переменную окружения
export GITLAB_TOKEN=glpat-...

# Логин
claudex auth login gitlab --profile gitlab-duo

# Проверить
claudex auth status
```

<Aside type="note">
  GitLab Duo требует переменную окружения `GITLAB_TOKEN` с Personal Access Token, имеющим scope `ai_features`. Для self-managed инстансов GitLab обновите `base_url` соответственно.
</Aside>

## Локальные модели

### Ollama

1. Установите Ollama: [ollama.com](https://ollama.com/)
2. Загрузите модель:

```bash
ollama pull qwen2.5:72b
```

3. Настройте Claudex:

```toml
[[profiles]]
name = "local-qwen"
provider_type = "OpenAICompatible"
base_url = "http://localhost:11434/v1"
api_key = ""
default_model = "qwen2.5:72b"
```

```bash
claudex profile test local-qwen
```

<Aside type="note">
  Ollama не требует API-ключа. Оставьте `api_key` пустым или не указывайте его.
</Aside>

### vLLM

1. Установите и запустите vLLM:

```bash
pip install vllm
vllm serve meta-llama/Llama-3.3-70B-Instruct --port 8000
```

2. Настройте Claudex:

```toml
[[profiles]]
name = "local-llama"
provider_type = "OpenAICompatible"
base_url = "http://localhost:8000/v1"
api_key = ""
default_model = "meta-llama/Llama-3.3-70B-Instruct"
```

### LM Studio

1. Скачайте LM Studio с [lmstudio.ai](https://lmstudio.ai/)
2. Загрузите модель и запустите локальный сервер (порт по умолчанию: 1234)

```toml
[[profiles]]
name = "lm-studio"
provider_type = "OpenAICompatible"
base_url = "http://localhost:1234/v1"
api_key = "lm-studio"
default_model = "local-model"
```

## Проверка настройки

После настройки любого провайдера проверьте подключение:

```bash
# Тестировать конкретный профиль
claudex profile test <profile-name>

# Тестировать все профили
claudex profile test all

# Список всех настроенных профилей
claudex profile list
```
