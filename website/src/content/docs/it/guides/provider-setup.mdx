Questa guida copre le istruzioni dettagliate per la configurazione di ogni fornitore supportato da Claudex, inclusi fornitori con chiave API, abbonamenti OAuth, piattaforme cloud e modelli locali.

## Fornitori con chiave API

### Anthropic

1. Registrati su [console.anthropic.com](https://console.anthropic.com/)
2. Vai su **API Keys** e crea una nuova chiave (formato: `sk-ant-*`)

```toml
[[profiles]]
name = "anthropic"
provider_type = "DirectAnthropic"
base_url = "https://api.anthropic.com"
api_key = "sk-ant-..."
default_model = "claude-sonnet-4-20250514"
```

```bash
claudex profile test anthropic
```

## Piattaforme cloud

### Azure OpenAI

1. Crea una risorsa Azure OpenAI nel [portale Azure](https://portal.azure.com/)
2. Distribuisci un modello e annota il nome della risorsa e del deployment
3. Ottieni la chiave API da **Chiavi e endpoint**

```toml
[[profiles]]
name = "azure-openai"
provider_type = "OpenAICompatible"
base_url = "https://YOUR_RESOURCE.openai.azure.com/openai/deployments/YOUR_DEPLOYMENT"
api_key = "YOUR_AZURE_KEY"
default_model = "gpt-4o"

[profiles.query_params]
api-version = "2024-12-01-preview"
```

```bash
claudex profile test azure-openai
```

<Aside type="note">
  Azure OpenAI viene rilevato automaticamente quando `base_url` contiene `openai.azure.com`. L'autenticazione utilizza l'header `api-key` al posto di `Authorization: Bearer`. Il parametro query `api-version` e obbligatorio.
</Aside>

### Google Vertex AI

1. Abilita l'API Vertex AI nel tuo progetto GCP
2. Genera un token di accesso con `gcloud auth print-access-token`

```toml
[[profiles]]
name = "vertex-ai"
provider_type = "DirectAnthropic"
base_url = "https://us-east5-aiplatform.googleapis.com/v1/projects/YOUR_PROJECT/locations/us-east5/publishers/anthropic/models"
api_key = "YOUR_GCLOUD_TOKEN"
default_model = "claude-sonnet-4@20250514"
```

```bash
claudex profile test vertex-ai
```

<Aside type="caution">
  L'output di `gcloud auth print-access-token` scade dopo circa 1 ora. Per l'uso in produzione, considera un service account con automazione del rinnovo del token.
</Aside>

### AWS Bedrock (tramite LiteLLM)

AWS Bedrock e supportato tramite un proxy LiteLLM:

1. Installa LiteLLM: `pip install litellm`
2. Avvia il proxy:

```bash
litellm --model bedrock/anthropic.claude-sonnet-4-20250514-v2:0
```

3. Configura Claudex:

```toml
[[profiles]]
name = "bedrock"
provider_type = "OpenAICompatible"
base_url = "http://localhost:4000/v1"
api_key = "sk-litellm"
default_model = "bedrock/anthropic.claude-sonnet-4-20250514-v2:0"
```

## Abbonamenti OAuth

L'autenticazione OAuth ti permette di usare abbonamenti esistenti (ChatGPT Plus, Claude Max, ecc.) invece di chiavi API separate.

### Claude Max / Pro

Usa il tuo abbonamento Claude esistente tramite la sessione OAuth nativa di Claude Code.

**Prerequisiti**: Claude Code installato e con accesso effettuato (il comando `claude` funziona normalmente)

```toml
[[profiles]]
name = "claude-max"
provider_type = "DirectAnthropic"
base_url = "https://api.claude.ai"
default_model = "claude-sonnet-4-20250514"
auth_type = "oauth"
oauth_provider = "claude"

[profiles.models]
haiku = "claude-haiku-4-20250514"
sonnet = "claude-sonnet-4-20250514"
opus = "claude-opus-4-20250514"
```

<Aside type="note">
  I profili OAuth di Claude usano una modalita gateway auth speciale. Claudex salta il proxy e lascia che Claude Code utilizzi il proprio OAuth direttamente. Non e necessario un passaggio `claudex auth login` separato.
</Aside>

### ChatGPT Plus / Codex CLI

Usa il tuo abbonamento ChatGPT Plus o Pro tramite il token Codex CLI.

**Prerequisiti**: Installa [Codex CLI](https://github.com/openai/codex) e autentica:

```bash
# Installa Codex CLI
npm install -g @openai/codex

# Autenticati (apre il browser)
codex auth
```

Poi configura Claudex:

```toml
[[profiles]]
name = "codex-sub"
provider_type = "OpenAIResponses"
base_url = "https://chatgpt.com/backend-api/codex"
default_model = "gpt-5.3-codex"
auth_type = "oauth"
oauth_provider = "openai"

[profiles.models]
haiku = "codex-mini-latest"
sonnet = "gpt-5.3-codex"
opus = "gpt-5.3-codex"
```

```bash
# Leggi il token da Codex CLI
claudex auth login openai --profile codex-sub

# Verifica
claudex auth status

# Esegui
claudex run codex-sub
```

<Aside type="tip">
  Modelli supportati sull'endpoint Codex: `gpt-5.3-codex`, `gpt-5.1-codex`, `codex-mini-latest`. Nota che `gpt-4o` NON e supportato sull'endpoint Codex di `chatgpt.com`. `strip_params = "auto"` rileva automaticamente questo endpoint e rimuove `temperature`/`top_p`/`top_k`.
</Aside>

### GitLab Duo

Usa il tuo abbonamento GitLab Duo tramite un Personal Access Token.

```toml
[[profiles]]
name = "gitlab-duo"
provider_type = "OpenAICompatible"
base_url = "https://gitlab.com/api/v4/ai/llm/proxy"
default_model = "claude-sonnet-4-20250514"
auth_type = "oauth"
oauth_provider = "gitlab"
```

```bash
# Imposta il token GitLab come variabile d'ambiente
export GITLAB_TOKEN=glpat-...

# Login
claudex auth login gitlab --profile gitlab-duo

# Verifica
claudex auth status
```

<Aside type="note">
  GitLab Duo richiede una variabile d'ambiente `GITLAB_TOKEN` con un Personal Access Token che abbia lo scope `ai_features`. Per istanze GitLab self-hosted, aggiorna `base_url` di conseguenza.
</Aside>

## Modelli locali

### Ollama

1. Installa Ollama: [ollama.com](https://ollama.com/)
2. Scarica un modello:

```bash
ollama pull qwen2.5:72b
```

3. Configura Claudex:

```toml
[[profiles]]
name = "local-qwen"
provider_type = "OpenAICompatible"
base_url = "http://localhost:11434/v1"
api_key = ""
default_model = "qwen2.5:72b"
```

```bash
claudex profile test local-qwen
```

<Aside type="note">
  Ollama non richiede una chiave API. Lascia `api_key` vuoto o omettilo.
</Aside>

### vLLM

1. Installa e avvia vLLM:

```bash
pip install vllm
vllm serve meta-llama/Llama-3.3-70B-Instruct --port 8000
```

2. Configura Claudex:

```toml
[[profiles]]
name = "local-llama"
provider_type = "OpenAICompatible"
base_url = "http://localhost:8000/v1"
api_key = ""
default_model = "meta-llama/Llama-3.3-70B-Instruct"
```

### LM Studio

1. Scarica LM Studio da [lmstudio.ai](https://lmstudio.ai/)
2. Carica un modello e avvia il server locale (porta predefinita: 1234)

```toml
[[profiles]]
name = "lm-studio"
provider_type = "OpenAICompatible"
base_url = "http://localhost:1234/v1"
api_key = "lm-studio"
default_model = "local-model"
```

## Verifica della configurazione

Dopo aver configurato un fornitore, verifica la connettivita:

```bash
# Testa un profilo specifico
claudex profile test <profile-name>

# Testa tutti i profili
claudex profile test all

# Elenca tutti i profili configurati
claudex profile list
```
