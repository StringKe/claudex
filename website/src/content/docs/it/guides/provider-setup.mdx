Questa guida copre le istruzioni dettagliate per la configurazione di ogni provider supportato da Claudex, inclusi provider con chiave API, abbonamenti OAuth e modelli locali.

## Provider con Chiave API

### Anthropic

1. Registrati su [console.anthropic.com](https://console.anthropic.com/)
2. Vai su **API Keys** e crea una nuova chiave (formato: `sk-ant-*`)

```toml
[[profiles]]
name = "anthropic"
provider_type = "DirectAnthropic"
base_url = "https://api.anthropic.com"
api_key = "sk-ant-..."
default_model = "claude-sonnet-4-20250514"
```

```bash
claudex profile test anthropic
```

### OpenRouter

1. Registrati su [openrouter.ai](https://openrouter.ai/)
2. Ottieni la tua chiave API su [openrouter.ai/keys](https://openrouter.ai/keys)

```toml
[[profiles]]

provider_type = "OpenAICompatible"
base_url = "https://openrouter.ai/api/v1"
api_key = "sk-or-..."
default_model = "anthropic/claude-sonnet-4"
```

```bash
claudex profile test openrouter
```

<Aside type="tip">
  OpenRouter supporta centinaia di modelli da diversi provider. Usa il prefisso `anthropic/` per i modelli Claude.
</Aside>

## Abbonamenti OAuth

L'autenticazione OAuth ti permette di usare abbonamenti provider esistenti (ChatGPT Plus, Claude Max, ecc.) invece di chiavi API separate.

### Claude Max / Pro

Usa il tuo abbonamento Claude esistente tramite la sessione OAuth nativa di Claude Code.

**Prerequisiti**: Claude Code installato e con accesso effettuato (il comando `claude` funziona normalmente)

```toml
[[profiles]]
name = "claude-sub"
provider_type = "DirectAnthropic"
base_url = "https://api.anthropic.com"
default_model = "claude-sonnet-4-20250514"
auth_type = "oauth"
oauth_provider = "claude"

[profiles.models]
haiku = "claude-haiku-4-20250514"
sonnet = "claude-sonnet-4-20250514"
opus = "claude-opus-4-20250514"
```

<Aside type="note">
  I profili OAuth di Claude usano una modalità gateway auth speciale. Claudex imposta `ANTHROPIC_AUTH_TOKEN` invece di `ANTHROPIC_API_KEY` per evitare conflitti con il login tramite abbonamento nativo di Claude Code. Il proxy viene saltato per i profili di abbonamento Claude.
</Aside>

Non è necessario un passaggio `claudex auth login` separato. Claudex legge la sessione esistente da `~/.claude`.

### ChatGPT Plus / Codex CLI

Usa il tuo abbonamento ChatGPT Plus o Pro tramite il token Codex CLI.

**Prerequisiti**: Installa [Codex CLI](https://github.com/openai/codex) e autentica:

```bash
# Installa Codex CLI
npm install -g @openai/codex

# Autenticati (apre il browser)
codex auth
```

Poi configura Claudex:

```toml
[[profiles]]
name = "codex-sub"
provider_type = "OpenAIResponses"
base_url = "https://chatgpt.com/backend-api/codex"
default_model = "gpt-4o"
auth_type = "oauth"
oauth_provider = "openai"

[profiles.models]
haiku = "gpt-4o-mini"
sonnet = "gpt-4o"
opus = "o1-pro"
```

```bash
# Leggi il token da Codex CLI
claudex auth login openai --profile codex-sub

# Verifica
claudex auth status

# Esegui
claudex run codex-sub
```

<Aside type="tip">
  Il provider OAuth `openai` usa `OpenAIResponses` come tipo di provider. Questo comunica tramite l'API OpenAI Responses (endpoint `/responses`) invece dell'API Chat Completions (`/chat/completions`).
</Aside>

### Google Gemini

Usa il tuo account Google tramite il flusso OAuth Device Code.

```toml
[[profiles]]
name = "gemini"
provider_type = "OpenAICompatible"
base_url = "https://generativelanguage.googleapis.com/v1beta/openai"
default_model = "gemini-2.5-pro"
auth_type = "oauth"
oauth_provider = "google"

[profiles.models]
haiku = "gemini-2.0-flash"
sonnet = "gemini-2.5-pro"
opus = "gemini-2.5-pro"
```

```bash
# Avvia il flusso device code (apre il browser per il login Google)
claudex auth login google --profile gemini

# Verifica
claudex auth status
```

## Modelli Locali

### Ollama

1. Installa Ollama: [ollama.com](https://ollama.com/)
2. Scarica un modello:

```bash
ollama pull qwen2.5:72b
```

3. Configura Claudex:

```toml
[[profiles]]
name = "local-qwen"
provider_type = "OpenAICompatible"
base_url = "http://localhost:11434/v1"
api_key = ""
default_model = "qwen2.5:72b"
```

```bash
claudex profile test local-qwen
```

<Aside type="note">
  Ollama non richiede una chiave API. Lascia `api_key` vuoto o omettilo.
</Aside>

### vLLM

1. Installa e avvia vLLM:

```bash
pip install vllm
vllm serve meta-llama/Llama-3.3-70B-Instruct --port 8000
```

2. Configura Claudex:

```toml
[[profiles]]
name = "local-llama"
provider_type = "OpenAICompatible"
base_url = "http://localhost:8000/v1"
api_key = ""
default_model = "meta-llama/Llama-3.3-70B-Instruct"
```

### LM Studio

1. Scarica LM Studio da [lmstudio.ai](https://lmstudio.ai/)
2. Carica un modello e avvia il server locale (porta predefinita: 1234)

```toml
[[profiles]]
name = "lm-studio"
provider_type = "OpenAICompatible"
base_url = "http://localhost:1234/v1"
api_key = ""
default_model = "loaded-model-name"
```

## Verifica della Configurazione

Dopo aver configurato un provider, verifica la connettività:

```bash
# Testa un profilo specifico
claudex profile test <nome-profilo>

# Testa tutti i profili
claudex profile test all

# Elenca tutti i profili configurati
claudex profile list
```
