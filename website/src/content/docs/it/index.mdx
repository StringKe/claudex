## Fornitori supportati

| Categoria | Fornitori |
|-----------|-----------|
| **Direct Anthropic** | Anthropic, MiniMax |
| **OpenAI Compatible** | Grok (xAI), OpenAI, DeepSeek, Kimi/Moonshot, GLM (Zhipu), OpenRouter, Groq, Mistral, Together AI, Perplexity, Cerebras |
| **Piattaforme cloud** | Azure OpenAI, Google Vertex AI |
| **Abbonamenti OAuth** | Claude Max/Pro, ChatGPT Plus/Pro (Codex), Google Gemini, GitHub Copilot, GitLab Duo, Qwen, Kimi |
| **Modelli locali** | Ollama, vLLM, LM Studio, llama.cpp |
| **Tramite LiteLLM** | AWS Bedrock e qualsiasi fornitore supportato da LiteLLM |

## Installazione rapida

```bash
curl -fsSL https://raw.githubusercontent.com/StringKe/claudex/main/install.sh | bash
```

## Come funziona

```
claudex run grok
    │
    ├── Start proxy → 127.0.0.1:13456
    │
    └── exec claude with:
        ANTHROPIC_BASE_URL=http://127.0.0.1:13456/proxy/grok
        ANTHROPIC_AUTH_TOKEN=<session-token>
        ANTHROPIC_MODEL=grok-3-beta
```

Il proxy traduce in modo trasparente tra i protocolli Anthropic e OpenAI, cosi Claude Code funziona senza problemi con qualsiasi fornitore supportato. Quando si utilizzano profili OAuth, viene impostato `ANTHROPIC_AUTH_TOKEN` (non `ANTHROPIC_API_KEY`) per evitare conflitti con il meccanismo di login dell'abbonamento nativo di Claude Code.
