## 전역 사용법

```
claudex [COMMAND]
```

명령 없이 실행하면, 프로파일이 설정된 경우 TUI 대시보드를 실행하고, 프로파일이 없으면 환영 메시지를 표시합니다.

## 명령

### `run`

특정 제공자 프로파일로 Claude Code를 실행합니다.

```bash
claudex run <PROFILE> [PROMPT] [-m MODEL] [--print] [--dangerously-skip-permissions] [ARGS...]
```

| 인수 | 설명 |
|----------|-------------|
| `PROFILE` | 프로파일 이름, 또는 스마트 라우팅을 위한 `auto` |
| `PROMPT` | 비대화형(단발성) 실행을 위한 선택적 프롬프트 |
| `-m, --model MODEL` | 이 세션의 기본 모델 오버라이드 |
| `--print` | 응답을 출력하고 종료 (비대화형 모드) |
| `--dangerously-skip-permissions` | 모든 권한 프롬프트 건너뛰기 (CI/CD용 `--print`와 함께 사용) |
| `ARGS...` | `claude`에 전달할 추가 인수 |

```bash
# Grok 사용 (대화형)
claudex run grok

# 특정 모델로 Grok 사용
claudex run grok -m grok-3-mini-beta

# 스마트 라우팅
claudex run auto

# 비대화형 단발성 실행
claudex run grok "Explain this codebase" --print

# CI/CD 파이프라인 사용 (프롬프트 없음, 권한 없음)
claudex run grok "Fix lint errors" --print --dangerously-skip-permissions

# claude에 추가 인수 전달
claudex run grok -- --verbose
```

### `profile`

제공자 프로파일을 관리합니다.

#### `profile list`

```bash
claudex profile list
```

상태와 함께 설정된 모든 프로파일을 나열합니다.

#### `profile add`

```bash
claudex profile add
```

새 프로파일을 추가하는 대화형 마법사. 제공자 선택, API 키 입력(선택적 키링 저장), 모델 선택, 연결 테스트를 안내합니다.

#### `profile show`

```bash
claudex profile show <NAME>
```

특정 프로파일의 세부 정보를 표시합니다.

#### `profile remove`

```bash
claudex profile remove <NAME>
```

설정에서 프로파일을 제거합니다.

#### `profile test`

```bash
claudex profile test <NAME|all>
```

특정 프로파일 또는 모든 프로파일의 연결을 테스트합니다.

### `auth`

제공자 프로파일의 OAuth 구독 인증을 관리합니다.

#### `auth login`

```bash
claudex auth login <PROVIDER>
```

OAuth를 사용하여 제공자에 로그인합니다. 지원 제공자: `claude`, `openai`, `google`, `qwen`, `kimi`, `github`.

**OpenAI**의 경우, Claudex가 Codex CLI 설정(`~/.codex/auth.json`)에서 토큰을 읽습니다. **Claude**의 경우, `~/.claude`에서 읽습니다. 다른 제공자는 로컬 콜백 서버 또는 device code 플로우를 사용합니다.

```bash
# OpenAI 로그인 (Codex CLI에서 읽기)
claudex auth login openai

# Claude 로그인 (~/.claude에서 읽기)
claudex auth login claude

# Google 로그인
claudex auth login google
```

#### `auth status`

```bash
claudex auth status [PROVIDER]
```

현재 인증 상태를 표시합니다. 제공자 인수 없이 실행하면 설정된 모든 OAuth 프로파일의 상태를 표시합니다.

```bash
# 모든 OAuth 프로파일 확인
claudex auth status

# 특정 제공자 확인
claudex auth status openai
```

#### `auth refresh`

```bash
claudex auth refresh <PROVIDER>
```

제공자의 OAuth 토큰을 수동으로 갱신합니다. 토큰은 만료 시 프록시에 의해 자동으로 갱신됩니다.

#### `auth logout`

```bash
claudex auth logout <PROVIDER>
```

제공자의 저장된 OAuth 토큰을 제거합니다.

### `proxy`

번역 프록시 서버를 관리합니다.

#### `proxy start`

```bash
claudex proxy start [-p PORT] [-d]
```

| 플래그 | 설명 |
|------|-------------|
| `-p, --port PORT` | 프록시 포트 오버라이드 (기본값: 설정에서 가져옴) |
| `-d, --daemon` | 백그라운드 데몬으로 실행 |

#### `proxy stop`

```bash
claudex proxy stop
```

프록시 데몬을 중지합니다 (SIGTERM 전송).

#### `proxy status`

```bash
claudex proxy status
```

프록시 실행 여부와 포트를 표시합니다.

### `dashboard`

```bash
claudex dashboard
```

실시간 모니터링 및 빠른 실행을 위한 TUI 대시보드를 실행합니다.

### `config`

```bash
claudex config [--init]
```

| 플래그 | 설명 |
|------|-------------|
| `--init` | 현재 디렉토리에 새 설정 파일 생성 |

`--init` 없이 실행하면 로드된 설정 파일 경로와 탐색 순서를 표시합니다.

### `update`

```bash
claudex update [--check]
```

| 플래그 | 설명 |
|------|-------------|
| `--check` | 업데이트만 확인하고 설치하지 않음 |

`--check` 없이 실행하면 최신 버전을 다운로드하고 설치합니다.
