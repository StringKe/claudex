Diese Anleitung enthaelt detaillierte Einrichtungsanweisungen fuer alle von Claudex unterstuetzten Anbieter, einschliesslich API-Schluessel-Anbieter, OAuth-Abonnements, Cloud-Plattformen und lokale Modelle.

## API-Schluessel-Anbieter

### Anthropic

1. Auf [console.anthropic.com](https://console.anthropic.com/) registrieren
2. Zu **API Keys** navigieren und einen neuen Schluessel erstellen (Format: `sk-ant-*`)

```toml
[[profiles]]
name = "anthropic"
provider_type = "DirectAnthropic"
base_url = "https://api.anthropic.com"
api_key = "sk-ant-..."
default_model = "claude-sonnet-4-20250514"
```

```bash
claudex profile test anthropic
```

## Cloud-Plattformen

### Azure OpenAI

1. Eine Azure OpenAI-Ressource im [Azure Portal](https://portal.azure.com/) erstellen
2. Ein Modell bereitstellen und den Ressourcennamen sowie den Bereitstellungsnamen notieren
3. Den API-Schluessel unter **Schluessel und Endpunkt** abrufen

```toml
[[profiles]]
name = "azure-openai"
provider_type = "OpenAICompatible"
base_url = "https://YOUR_RESOURCE.openai.azure.com/openai/deployments/YOUR_DEPLOYMENT"
api_key = "YOUR_AZURE_KEY"
default_model = "gpt-4o"

[profiles.query_params]
api-version = "2024-12-01-preview"
```

```bash
claudex profile test azure-openai
```

<Aside type="note">
  Azure OpenAI wird automatisch erkannt, wenn `base_url` den String `openai.azure.com` enthaelt. Die Authentifizierung verwendet den `api-key`-Header anstelle von `Authorization: Bearer`. Der `api-version`-Abfrageparameter ist erforderlich.
</Aside>

### Google Vertex AI

1. Die Vertex AI API in Ihrem GCP-Projekt aktivieren
2. Ein Access Token mit `gcloud auth print-access-token` generieren

```toml
[[profiles]]
name = "vertex-ai"
provider_type = "DirectAnthropic"
base_url = "https://us-east5-aiplatform.googleapis.com/v1/projects/YOUR_PROJECT/locations/us-east5/publishers/anthropic/models"
api_key = "YOUR_GCLOUD_TOKEN"
default_model = "claude-sonnet-4@20250514"
```

```bash
claudex profile test vertex-ai
```

<Aside type="caution">
  Die Ausgabe von `gcloud auth print-access-token` laeuft nach ca. 1 Stunde ab. Fuer den Produktiveinsatz ein Dienstkonto mit automatischer Token-Aktualisierung in Betracht ziehen.
</Aside>

### AWS Bedrock (ueber LiteLLM)

AWS Bedrock wird ueber einen LiteLLM-Proxy unterstuetzt:

1. LiteLLM installieren: `pip install litellm`
2. Den Proxy starten:

```bash
litellm --model bedrock/anthropic.claude-sonnet-4-20250514-v2:0
```

3. Claudex konfigurieren:

```toml
[[profiles]]
name = "bedrock"
provider_type = "OpenAICompatible"
base_url = "http://localhost:4000/v1"
api_key = "sk-litellm"
default_model = "bedrock/anthropic.claude-sonnet-4-20250514-v2:0"
```

## OAuth-Abonnements

Die OAuth-Authentifizierung ermoeglicht die Verwendung vorhandener Anbieterabonnements (ChatGPT Plus, Claude Max usw.) anstelle separater API-Schluessel.

### Claude Max / Pro

Das vorhandene Claude-Abonnement ueber die native OAuth-Sitzung von Claude Code verwenden.

**Voraussetzungen**: Claude Code installiert und angemeldet (der `claude`-Befehl funktioniert normal)

```toml
[[profiles]]
name = "claude-max"
provider_type = "DirectAnthropic"
base_url = "https://api.claude.ai"
default_model = "claude-sonnet-4-20250514"
auth_type = "oauth"
oauth_provider = "claude"

[profiles.models]
haiku = "claude-haiku-4-20250514"
sonnet = "claude-sonnet-4-20250514"
opus = "claude-opus-4-20250514"
```

<Aside type="note">
  Claude OAuth-Profile verwenden einen speziellen Gateway-Auth-Modus. Claudex ueberspringt den Proxy und laesst Claude Code sein eigenes OAuth direkt verwenden. Kein separater `claudex auth login`-Schritt erforderlich.
</Aside>

### ChatGPT Plus / Codex CLI

Das ChatGPT Plus- oder Pro-Abonnement ueber den Codex CLI-Token verwenden.

**Voraussetzungen**: [Codex CLI](https://github.com/openai/codex) installieren und authentifizieren:

```bash
# Codex CLI installieren
npm install -g @openai/codex

# Authentifizieren (oeffnet Browser)
codex auth
```

Dann Claudex konfigurieren:

```toml
[[profiles]]
name = "codex-sub"
provider_type = "OpenAIResponses"
base_url = "https://chatgpt.com/backend-api/codex"
default_model = "gpt-5.3-codex"
auth_type = "oauth"
oauth_provider = "openai"

[profiles.models]
haiku = "codex-mini-latest"
sonnet = "gpt-5.3-codex"
opus = "gpt-5.3-codex"
```

```bash
# Token aus Codex CLI lesen
claudex auth login openai --profile codex-sub

# Ueberpruefen
claudex auth status

# Ausfuehren
claudex run codex-sub
```

<Aside type="tip">
  Unterstuetzte Modelle auf dem Codex-Endpunkt: `gpt-5.3-codex`, `gpt-5.1-codex`, `codex-mini-latest`. Beachten Sie, dass `gpt-4o` auf dem `chatgpt.com`-Codex-Endpunkt NICHT unterstuetzt wird. `strip_params = "auto"` erkennt diesen Endpunkt automatisch und entfernt `temperature`/`top_p`/`top_k`.
</Aside>

### GitLab Duo

GitLab Duo-Abonnement ueber einen Personal Access Token verwenden.

```toml
[[profiles]]
name = "gitlab-duo"
provider_type = "OpenAICompatible"
base_url = "https://gitlab.com/api/v4/ai/llm/proxy"
default_model = "claude-sonnet-4-20250514"
auth_type = "oauth"
oauth_provider = "gitlab"
```

```bash
# GitLab-Token als Umgebungsvariable setzen
export GITLAB_TOKEN=glpat-...

# Login
claudex auth login gitlab --profile gitlab-duo

# Ueberpruefen
claudex auth status
```

<Aside type="note">
  GitLab Duo erfordert eine `GITLAB_TOKEN`-Umgebungsvariable mit einem Personal Access Token, der den `ai_features`-Scope hat. Fuer selbst-gehostete GitLab-Instanzen `base_url` entsprechend anpassen.
</Aside>

## Lokale Modelle

### Ollama

1. Ollama installieren: [ollama.com](https://ollama.com/)
2. Ein Modell herunterladen:

```bash
ollama pull qwen2.5:72b
```

3. Claudex konfigurieren:

```toml
[[profiles]]
name = "local-qwen"
provider_type = "OpenAICompatible"
base_url = "http://localhost:11434/v1"
api_key = ""
default_model = "qwen2.5:72b"
```

```bash
claudex profile test local-qwen
```

<Aside type="note">
  Ollama benoetigt keinen API-Schluessel. `api_key` leer lassen oder weglassen.
</Aside>

### vLLM

1. vLLM installieren und starten:

```bash
pip install vllm
vllm serve meta-llama/Llama-3.3-70B-Instruct --port 8000
```

2. Claudex konfigurieren:

```toml
[[profiles]]
name = "local-llama"
provider_type = "OpenAICompatible"
base_url = "http://localhost:8000/v1"
api_key = ""
default_model = "meta-llama/Llama-3.3-70B-Instruct"
```

### LM Studio

1. LM Studio von [lmstudio.ai](https://lmstudio.ai/) herunterladen
2. Ein Modell laden und den lokalen Server starten (Standardport: 1234)

```toml
[[profiles]]
name = "lm-studio"
provider_type = "OpenAICompatible"
base_url = "http://localhost:1234/v1"
api_key = "lm-studio"
default_model = "local-model"
```

## Einrichtung ueberpruefen

Nach der Konfiguration eines Anbieters die Verbindung ueberpruefen:

```bash
# Ein bestimmtes Profil testen
claudex profile test <profile-name>

# Alle Profile testen
claudex profile test all

# Alle konfigurierten Profile auflisten
claudex profile list
```
