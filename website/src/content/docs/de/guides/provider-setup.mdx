Diese Anleitung enthält detaillierte Einrichtungsanweisungen für alle von Claudex unterstützten Anbieter, einschließlich API-Schlüssel-Anbietern, OAuth-Abonnements und lokalen Modellen.

## API-Schlüssel-Anbieter

### Anthropic

1. Auf [console.anthropic.com](https://console.anthropic.com/) registrieren
2. Zu **API Keys** navigieren und einen neuen Schlüssel erstellen (Format: `sk-ant-*`)

```toml
[[profiles]]
name = "anthropic"
provider_type = "DirectAnthropic"
base_url = "https://api.anthropic.com"
api_key = "sk-ant-..."
default_model = "claude-sonnet-4-20250514"
```

```bash
claudex profile test anthropic
```

### MiniMax

1. Auf [platform.minimaxi.com](https://platform.minimaxi.com/) registrieren
2. In der Konsole zu **API Keys** navigieren

```toml
[[profiles]]
```

### OpenRouter

1. Auf [openrouter.ai](https://openrouter.ai/) registrieren
2. API-Schlüssel unter [openrouter.ai/keys](https://openrouter.ai/keys) abrufen

```toml
[[profiles]]

base_url = "https://openrouter.ai/api/v1"
```

```bash
claudex profile test openrouter
```

<Aside type="tip">
  OpenRouter unterstützt Hunderte von Modellen verschiedener Anbieter. Das Präfix `anthropic/` für Claude-Modelle verwenden.
</Aside>

## OAuth-Abonnements

Die OAuth-Authentifizierung ermöglicht die Verwendung vorhandener Anbieterabonnements (ChatGPT Plus, Claude Max usw.) anstelle separater API-Schlüssel.

### Claude Max / Pro

Das vorhandene Claude-Abonnement über die native OAuth-Sitzung von Claude Code verwenden.

**Voraussetzungen**: Claude Code installiert und angemeldet (der `claude`-Befehl funktioniert normal)

```toml
[[profiles]]
name = "claude-sub"
provider_type = "DirectAnthropic"
base_url = "https://api.anthropic.com"
default_model = "claude-sonnet-4-20250514"
auth_type = "oauth"
oauth_provider = "claude"

[profiles.models]
haiku = "claude-haiku-4-20250514"
sonnet = "claude-sonnet-4-20250514"
opus = "claude-opus-4-20250514"
```

<Aside type="note">
  Claude OAuth-Profile verwenden einen speziellen Gateway-Authentifizierungsmodus. Claudex setzt `ANTHROPIC_AUTH_TOKEN` statt `ANTHROPIC_API_KEY`, um Konflikte mit dem eigenen Abonnement-Login von Claude Code zu vermeiden. Der Proxy wird für Claude-Abonnementprofile übersprungen.
</Aside>

Kein separater `claudex auth login`-Schritt erforderlich. Claudex liest die vorhandene Sitzung aus `~/.claude`.

### ChatGPT Plus / Codex CLI

Das ChatGPT Plus- oder Pro-Abonnement über den Codex CLI-Token verwenden.

**Voraussetzungen**: [Codex CLI](https://github.com/openai/codex) installieren und authentifizieren:

```bash
# Codex CLI installieren
npm install -g @openai/codex

# Authentifizieren (öffnet Browser)
codex auth
```

Dann Claudex konfigurieren:

```toml
[[profiles]]
name = "codex-sub"
provider_type = "OpenAIResponses"
base_url = "https://chatgpt.com/backend-api/codex"
default_model = "gpt-4o"
auth_type = "oauth"
oauth_provider = "openai"

[profiles.models]
haiku = "gpt-4o-mini"
sonnet = "gpt-4o"
opus = "o1-pro"
```

```bash
# Token aus Codex CLI lesen
claudex auth login openai --profile codex-sub

# Überprüfen
claudex auth status

# Ausführen
claudex run codex-sub
```

<Aside type="tip">
  Der `openai` OAuth-Anbieter verwendet `OpenAIResponses` als Anbietertyp. Dies kommuniziert über die OpenAI Responses API (`/responses`-Endpunkt) statt über die Chat Completions API (`/chat/completions`).
</Aside>

### Google Gemini

Das Google-Konto über den OAuth Device Code Flow verwenden.

```toml
[[profiles]]
name = "gemini"
provider_type = "OpenAICompatible"
base_url = "https://generativelanguage.googleapis.com/v1beta/openai"
default_model = "gemini-2.5-pro"
auth_type = "oauth"
oauth_provider = "google"

[profiles.models]
haiku = "gemini-2.0-flash"
sonnet = "gemini-2.5-pro"
opus = "gemini-2.5-pro"
```

```bash
# Device Code Flow starten (öffnet Browser für Google-Anmeldung)
claudex auth login google --profile gemini

# Überprüfen
claudex auth status
```

## Lokale Modelle

### Ollama

1. Ollama installieren: [ollama.com](https://ollama.com/)
2. Ein Modell herunterladen:

```bash
ollama pull qwen2.5:72b
```

3. Claudex konfigurieren:

```toml
[[profiles]]
name = "local-qwen"
provider_type = "OpenAICompatible"
base_url = "http://localhost:11434/v1"
api_key = ""
default_model = "qwen2.5:72b"
```

```bash
claudex profile test local-qwen
```

<Aside type="note">
  Ollama benötigt keinen API-Schlüssel. `api_key` leer lassen oder weglassen.
</Aside>

### vLLM

1. vLLM installieren und starten:

```bash
pip install vllm
vllm serve meta-llama/Llama-3.3-70B-Instruct --port 8000
```

2. Claudex konfigurieren:

```toml
[[profiles]]
name = "local-llama"
provider_type = "OpenAICompatible"
base_url = "http://localhost:8000/v1"
api_key = ""
default_model = "meta-llama/Llama-3.3-70B-Instruct"
```

### LM Studio

1. LM Studio von [lmstudio.ai](https://lmstudio.ai/) herunterladen
2. Ein Modell laden und den lokalen Server starten (Standardport: 1234)

```toml
[[profiles]]
name = "lm-studio"
provider_type = "OpenAICompatible"
base_url = "http://localhost:1234/v1"
api_key = ""
default_model = "loaded-model-name"
```

## Einrichtung überprüfen

Nach der Konfiguration eines Anbieters die Verbindung überprüfen:

```bash
# Ein bestimmtes Profil testen
claudex profile test <profile-name>

# Alle Profile testen
claudex profile test all

# Alle konfigurierten Profile auflisten
claudex profile list
```
