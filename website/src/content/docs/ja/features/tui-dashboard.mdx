---
title: TUI ダッシュボード
description: プロファイルの監視と管理のためのリアルタイムターミナルダッシュボード
---

TUI ダッシュボードはプロファイルの健全性の監視、メトリクスの確認、Claude Code セッションのクイック起動のためのリアルタイムターミナルインターフェイスを提供します。

## 起動

```bash
# 明示的に起動
claudex dashboard

# 自動起動 (プロファイルが存在し、サブコマンドが指定されていない場合)
claudex
```

## キーボードショートカット

| キー | 操作 |
|------|------|
| `j` / `↓` | 選択を下へ移動 |
| `k` / `↑` | 選択を上へ移動 |
| `Enter` | 選択したプロファイルで Claude Code を起動 |
| `t` | 選択したプロファイルの接続テスト |
| `a` | 新しいプロファイルを追加 (ヒントを表示) |
| `d` | 選択したプロファイルを削除 (ヒントを表示) |
| `p` | プロキシの起動/停止を切り替え |
| `/` | 検索モードに入る |
| `Esc` | 検索モードを終了 / ヘルプを閉じる |
| `?` | ヘルプポップアップを表示 |
| `q` / `Ctrl+C` | ダッシュボードを終了 |

## ダッシュボードのレイアウト

ダッシュボードに表示される内容:

- **プロファイルリスト** — 名前、モデル、プロバイダータイプ、ヘルスステータスを含む設定済みのすべてのプロファイル
- **ヘルスインジケーター** — カラーコーディングされたステータス (緑 = 正常、黄 = 低下、赤 = サーキットブレーカーオープン)
- **メトリクスパネル** — プロファイルごとのリクエスト数、成功率、平均レイテンシ
- **ログパネル** — リアルタイムスクロールのログ出力

## ヘルスステータス

ダッシュボードはバックグラウンドヘルスチェッカーを実行し、約30秒ごとに有効なすべてのプロファイルに ping を送信します。ヘルスステータスはリアルタイムで反映されます:

| 色 | 意味 |
|----|------|
| 緑 | プロバイダーは正常に応答している |
| 黄 | 最近の失敗が検出された |
| 赤 | サーキットブレーカーがオープン |
| 灰 | プロファイルが無効 |
