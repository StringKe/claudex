Ten przewodnik zawiera szczegółowe instrukcje konfiguracji dla każdego dostawcy obsługiwanego przez Claudex, w tym dostawców kluczy API, subskrypcji OAuth i modeli lokalnych.

## Dostawcy kluczy API

### Anthropic

1. Zarejestruj się na [console.anthropic.com](https://console.anthropic.com/)
2. Przejdź do **API Keys** i utwórz nowy klucz (format: `sk-ant-*`)

```toml
[[profiles]]
name = "anthropic"
provider_type = "DirectAnthropic"
base_url = "https://api.anthropic.com"
api_key = "sk-ant-..."
default_model = "claude-sonnet-4-20250514"
```

```bash
claudex profile test anthropic
```

### MiniMax

1. Zarejestruj się na [platform.minimaxi.com](https://platform.minimaxi.com/)
2. Przejdź do **API Keys** w konsoli

```toml
[[profiles]]

api_key = "..."
```

### OpenRouter

1. Zarejestruj się na [openrouter.ai](https://openrouter.ai/)
2. Pobierz klucz API na [openrouter.ai/keys](https://openrouter.ai/keys)

```toml
[[profiles]]

provider_type = "OpenAICompatible"
base_url = "https://openrouter.ai/api/v1"
api_key = "sk-or-..."
```

```bash
claudex profile test openrouter
```

<Aside type="tip">
  OpenRouter obsługuje setki modeli od różnych dostawców. Użyj prefiksu `anthropic/` dla modeli Claude.
</Aside>

## Subskrypcje OAuth

Uwierzytelnianie OAuth pozwala używać istniejących subskrypcji dostawcy (ChatGPT Plus, Claude Max itp.) zamiast osobnych kluczy API.

### Claude Max / Pro

Użyj istniejącej subskrypcji Claude przez natywną sesję OAuth Claude Code.

**Wymagania wstępne**: Claude Code zainstalowany i zalogowany (polecenie `claude` działa normalnie)

```toml
[[profiles]]
name = "claude-sub"
provider_type = "DirectAnthropic"
base_url = "https://api.anthropic.com"
default_model = "claude-sonnet-4-20250514"
auth_type = "oauth"
oauth_provider = "claude"

[profiles.models]
haiku = "claude-haiku-4-20250514"
sonnet = "claude-sonnet-4-20250514"
opus = "claude-opus-4-20250514"
```

<Aside type="note">
  Profile OAuth Claude używają specjalnego trybu uwierzytelniania bramkowego. Claudex ustawia `ANTHROPIC_AUTH_TOKEN` zamiast `ANTHROPIC_API_KEY`, aby uniknąć konfliktu z własnym logowaniem subskrypcyjnym Claude Code. Proxy jest pomijane dla profili subskrypcji Claude.
</Aside>

Nie jest potrzebny osobny krok `claudex auth login`. Claudex odczytuje istniejącą sesję z `~/.claude`.

### ChatGPT Plus / Codex CLI

Użyj subskrypcji ChatGPT Plus lub Pro przez token Codex CLI.

**Wymagania wstępne**: Zainstaluj [Codex CLI](https://github.com/openai/codex) i uwierzytelnij się:

```bash
# Zainstaluj Codex CLI
npm install -g @openai/codex

# Uwierzytelnianie (otwiera przeglądarkę)
codex auth
```

Następnie skonfiguruj Claudex:

```toml
[[profiles]]
name = "codex-sub"
provider_type = "OpenAIResponses"
base_url = "https://chatgpt.com/backend-api/codex"
default_model = "gpt-4o"
auth_type = "oauth"
oauth_provider = "openai"

[profiles.models]
haiku = "gpt-4o-mini"
sonnet = "gpt-4o"
opus = "o1-pro"
```

```bash
# Odczytaj token z Codex CLI
claudex auth login openai --profile codex-sub

# Weryfikacja
claudex auth status

# Uruchomienie
claudex run codex-sub
```

<Aside type="tip">
  Dostawca OAuth `openai` używa `OpenAIResponses` jako typu dostawcy. Komunikuje się przez OpenAI Responses API (endpoint `/responses`) zamiast Chat Completions API (`/chat/completions`).
</Aside>

### Google Gemini

Użyj konta Google przez przepływ OAuth Device Code.

```toml
[[profiles]]
name = "gemini"
provider_type = "OpenAICompatible"
base_url = "https://generativelanguage.googleapis.com/v1beta/openai"
default_model = "gemini-2.5-pro"
auth_type = "oauth"
oauth_provider = "google"

[profiles.models]
haiku = "gemini-2.0-flash"
sonnet = "gemini-2.5-pro"
opus = "gemini-2.5-pro"
```

```bash
# Uruchom przepływ device code (otwiera przeglądarkę do logowania Google)
claudex auth login google --profile gemini

# Weryfikacja
claudex auth status
```

## Modele lokalne

### Ollama

1. Zainstaluj Ollama: [ollama.com](https://ollama.com/)
2. Pobierz model:

```bash
ollama pull qwen2.5:72b
```

3. Skonfiguruj Claudex:

```toml
[[profiles]]
name = "local-qwen"
provider_type = "OpenAICompatible"
base_url = "http://localhost:11434/v1"
api_key = ""
default_model = "qwen2.5:72b"
```

```bash
claudex profile test local-qwen
```

<Aside type="note">
  Ollama nie wymaga klucza API. Pozostaw `api_key` puste lub pomiń je.
</Aside>

### vLLM

1. Zainstaluj i uruchom vLLM:

```bash
pip install vllm
vllm serve meta-llama/Llama-3.3-70B-Instruct --port 8000
```

2. Skonfiguruj Claudex:

```toml
[[profiles]]
name = "local-llama"
provider_type = "OpenAICompatible"
base_url = "http://localhost:8000/v1"
api_key = ""
default_model = "meta-llama/Llama-3.3-70B-Instruct"
```

### LM Studio

1. Pobierz LM Studio z [lmstudio.ai](https://lmstudio.ai/)
2. Załaduj model i uruchom serwer lokalny (domyślny port: 1234)

```toml
[[profiles]]
name = "lm-studio"
provider_type = "OpenAICompatible"
base_url = "http://localhost:1234/v1"
api_key = ""
default_model = "loaded-model-name"
```

## Weryfikacja konfiguracji

Po skonfigurowaniu dowolnego dostawcy sprawdź łączność:

```bash
# Przetestuj konkretny profil
claudex profile test <profile-name>

# Przetestuj wszystkie profile
claudex profile test all

# Wylistuj wszystkie skonfigurowane profile
claudex profile list
```
