Ten przewodnik zawiera szczegolowe instrukcje konfiguracji dla kazdego dostawcy obslugiwanego przez Claudex, w tym dostawcow kluczy API, subskrypcji OAuth, platform chmurowych i modeli lokalnych.

## Dostawcy kluczy API

### Anthropic

1. Zarejestruj sie na [console.anthropic.com](https://console.anthropic.com/)
2. Przejdz do **API Keys** i utworz nowy klucz (format: `sk-ant-*`)

```toml
[[profiles]]
name = "anthropic"
provider_type = "DirectAnthropic"
base_url = "https://api.anthropic.com"
api_key = "sk-ant-..."
default_model = "claude-sonnet-4-20250514"
```

```bash
claudex profile test anthropic
```

## Platformy chmurowe

### Azure OpenAI

1. Utworz zasob Azure OpenAI w [Azure Portal](https://portal.azure.com/)
2. Wdraz model i zapisz nazwe zasobu oraz nazwe wdrozenia
3. Pobierz klucz API z **Keys and Endpoint**

```toml
[[profiles]]
name = "azure-openai"
provider_type = "OpenAICompatible"
base_url = "https://YOUR_RESOURCE.openai.azure.com/openai/deployments/YOUR_DEPLOYMENT"
api_key = "YOUR_AZURE_KEY"
default_model = "gpt-4o"

[profiles.query_params]
api-version = "2024-12-01-preview"
```

```bash
claudex profile test azure-openai
```

<Aside type="note">
  Azure OpenAI jest automatycznie wykrywany po `base_url` zawierajacym `openai.azure.com`. Uwierzytelnianie uzywa naglowka `api-key` zamiast `Authorization: Bearer`. Parametr zapytania `api-version` jest wymagany.
</Aside>

### Google Vertex AI

1. Wlacz API Vertex AI w swoim projekcie GCP
2. Wygeneruj token dostepu za pomoca `gcloud auth print-access-token`

```toml
[[profiles]]
name = "vertex-ai"
provider_type = "DirectAnthropic"
base_url = "https://us-east5-aiplatform.googleapis.com/v1/projects/YOUR_PROJECT/locations/us-east5/publishers/anthropic/models"
api_key = "YOUR_GCLOUD_TOKEN"
default_model = "claude-sonnet-4@20250514"
```

```bash
claudex profile test vertex-ai
```

<Aside type="caution">
  Wyjscie `gcloud auth print-access-token` wygasa po okolo 1 godzinie. Do uzycia produkcyjnego rozwazyc konto uslugowe z automatycznym odswiezaniem tokenow.
</Aside>

### AWS Bedrock (przez LiteLLM)

AWS Bedrock jest obslugiwany przez proxy LiteLLM:

1. Zainstaluj LiteLLM: `pip install litellm`
2. Uruchom proxy:

```bash
litellm --model bedrock/anthropic.claude-sonnet-4-20250514-v2:0
```

3. Skonfiguruj Claudex:

```toml
[[profiles]]
name = "bedrock"
provider_type = "OpenAICompatible"
base_url = "http://localhost:4000/v1"
api_key = "sk-litellm"
default_model = "bedrock/anthropic.claude-sonnet-4-20250514-v2:0"
```

## Subskrypcje OAuth

Uwierzytelnianie OAuth pozwala uzywac istniejacych subskrypcji dostawcy (ChatGPT Plus, Claude Max itp.) zamiast osobnych kluczy API.

### Claude Max / Pro

Uzyj istniejacej subskrypcji Claude przez natywna sesje OAuth Claude Code.

**Wymagania wstepne**: Claude Code zainstalowany i zalogowany (polecenie `claude` dziala normalnie)

```toml
[[profiles]]
name = "claude-max"
provider_type = "DirectAnthropic"
base_url = "https://api.claude.ai"
default_model = "claude-sonnet-4-20250514"
auth_type = "oauth"
oauth_provider = "claude"

[profiles.models]
haiku = "claude-haiku-4-20250514"
sonnet = "claude-sonnet-4-20250514"
opus = "claude-opus-4-20250514"
```

<Aside type="note">
  Profile OAuth Claude uzywaja specjalnego trybu uwierzytelniania bramkowego. Claudex pomija proxy i pozwala Claude Code uzywac wlasnego OAuth bezposrednio. Nie jest potrzebny osobny krok `claudex auth login`.
</Aside>

### ChatGPT Plus / Codex CLI

Uzyj subskrypcji ChatGPT Plus lub Pro przez token Codex CLI.

**Wymagania wstepne**: Zainstaluj [Codex CLI](https://github.com/openai/codex) i uwierzytelnij sie:

```bash
# Zainstaluj Codex CLI
npm install -g @openai/codex

# Uwierzytelnianie (otwiera przegladarke)
codex auth
```

Nastepnie skonfiguruj Claudex:

```toml
[[profiles]]
name = "codex-sub"
provider_type = "OpenAIResponses"
base_url = "https://chatgpt.com/backend-api/codex"
default_model = "gpt-5.3-codex"
auth_type = "oauth"
oauth_provider = "openai"

[profiles.models]
haiku = "codex-mini-latest"
sonnet = "gpt-5.3-codex"
opus = "gpt-5.3-codex"
```

```bash
# Odczytaj token z Codex CLI
claudex auth login openai --profile codex-sub

# Weryfikacja
claudex auth status

# Uruchomienie
claudex run codex-sub
```

<Aside type="tip">
  Obslugiwane modele na endpoincie Codex: `gpt-5.3-codex`, `gpt-5.1-codex`, `codex-mini-latest`. Model `gpt-4o` NIE jest obslugiwany na endpoincie `chatgpt.com` Codex. `strip_params = "auto"` automatycznie wykryje ten endpoint i usunie `temperature`/`top_p`/`top_k`.
</Aside>

```bash
# Ustaw token GitLab jako zmienna srodowiskowa

# Weryfikacja
claudex auth status
```

<Aside type="note">
  GitLab Duo wymaga zmiennej srodowiskowej `GITLAB_TOKEN` z Personal Access Token majacym scope `ai_features`. Dla samodzielnie hostowanych instancji GitLab zaktualizuj odpowiednio `base_url`.
</Aside>

## Modele lokalne

### Ollama

1. Zainstaluj Ollama: [ollama.com](https://ollama.com/)
2. Pobierz model:

```bash
ollama pull qwen2.5:72b
```

3. Skonfiguruj Claudex:

```toml
[[profiles]]
name = "local-qwen"
provider_type = "OpenAICompatible"
base_url = "http://localhost:11434/v1"
api_key = ""
default_model = "qwen2.5:72b"
```

```bash
claudex profile test local-qwen
```

<Aside type="note">
  Ollama nie wymaga klucza API. Pozostaw `api_key` puste lub pomin je.
</Aside>

### vLLM

1. Zainstaluj i uruchom vLLM:

```bash
pip install vllm
vllm serve meta-llama/Llama-3.3-70B-Instruct --port 8000
```

2. Skonfiguruj Claudex:

```toml
[[profiles]]
name = "local-llama"
provider_type = "OpenAICompatible"
base_url = "http://localhost:8000/v1"
api_key = ""
default_model = "meta-llama/Llama-3.3-70B-Instruct"
```

### LM Studio

1. Pobierz LM Studio z [lmstudio.ai](https://lmstudio.ai/)
2. Zaladuj model i uruchom serwer lokalny (domyslny port: 1234)

```toml
[[profiles]]
name = "lm-studio"
provider_type = "OpenAICompatible"
base_url = "http://localhost:1234/v1"
api_key = "lm-studio"
default_model = "local-model"
```

## Weryfikacja konfiguracji

Po skonfigurowaniu dowolnego dostawcy sprawdz lacznosc:

```bash
# Przetestuj konkretny profil
claudex profile test <profile-name>

# Przetestuj wszystkie profile
claudex profile test all

# Wylistuj wszystkie skonfigurowane profile
claudex profile list
```
